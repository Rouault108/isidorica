## 1. 予測誤差による学習駆動

**理論基盤**: 予測処理と自由エネルギー原理（1.1）、認知的不協和、生産的失敗

### 設計原則

学習の駆動力は**予測誤差**である。学習者が「知っているつもり（低い予測誤差）」のままでは学習は起きない。

- Phase 1の「Laboratory」は、学習者の素朴なメンタルモデルに対して意図的に**適度な**予測誤差（驚き）を生成し、脳を可塑的な「学習モード」へ移行させるための装置として機能する
- 予測誤差が大きすぎると理解不能となり学習が阻害されるため、学習者が認知的努力で解消可能な範囲（ZPD）に収める必要がある
- 誤概念を「消す」のではなく、科学的体系の中への再配置を促す

> **参照**: USER_BEHAVIOR_MODEL.md セクション3.1における行動予測への適用

### 教育実践フレームワーク

#### 1.1 POE法（Predict-Observe-Explain）

[White & Gunstone 1992]

| フェーズ | 内容 | Isidoricaでの実装 |
| :--- | :--- | :--- |
| **Predict（予測）** | 学習者が現象について予測を立て、理由を言語化 | Laboratory操作前の予測入力フィールド |
| **Observe（観察）** | 実験やシミュレーションで実際の結果を観察 | Laboratory/シミュレーションの実行 |
| **Explain（説明）** | 予測と観察のズレを説明し、理解を修正 | 記事本文での理論的説明、自己説明プロンプト |

POE法の効果は、予測誤差を**意識的かつ言語化された形**で体験させることにある。予測を明示化することで、学習者は自らの事前知識（Prior Model）を意識し、観察結果との乖離を「自分ごと」として認識する。

#### 1.2 生産的失敗（Productive Failure）

[Kapur 2008]

正解を教える前に、意図的に試行錯誤させることで「高い自由エネルギー」を経験させる。その後の解説が「誤差を一気に解消する特効薬」として機能し、深い理解を促進する。

| 要素 | 内容 | Isidoricaでの実装 |
| :--- | :--- | :--- |
| **問題提示** | 解法を教える前に課題を提示 | Phase 1: 定義前に問いを投げかける |
| **試行錯誤** | 学習者が自力で解こうとして失敗 | Laboratory: 自由探索モード |
| **解説** | 失敗後に概念を導入 | Phase 2: 形式的定義とモデルの更新 |

> **例（統計学）**: 「標準偏差」の公式を教える前に、「バラつきを数値で表す方法を考えてみよう」と投げかける。試行錯誤の後に公式を教えることで、「なぜ2乗するのか」が深く定着する。

#### 1.3 認知的不協和とソクラテス式問答

[Festinger 1957; Piaget 1977]

外部入力との誤差ではなく、学習者の**内部モデル間の矛盾（不整合）**を顕在化させる。信念と論理のコンフリクトにより、より高次元なモデルへの更新を促す。

| 要素 | 内容 | Isidoricaでの実装 |
| :--- | :--- | :--- |
| **初期信念の表明** | 学習者の素朴な信念を明示 | 記事冒頭での「よくある誤解」提示 |
| **反例の提示** | 信念と矛盾する事例を示す | LAB-DECISION: 思考実験シミュレーター |
| **統合** | より複雑なモデルへ更新 | 記事本文での概念的統合 |

> **例（倫理学）**: 「嘘は常に悪い」という信念に対し、「殺人犯に追われている友人の居場所を聞かれたら？」という反例を提示。

#### 1.4 即時フィードバック

[Hattie & Timperley 2007]

行動直後のフィードバックにより、**マイクロレベルの予測誤差**を高速で修正するサイクルを回す。フィードバックが遅れると、脳は「どの予測が間違っていたか」を特定できず学習効率が低下する。

| 要素 | Isidoricaでの実装 |
| :--- | :--- |
| **即時実行** | LAB-CODE: 書いた瞬間にエラー表示、実行結果即時表示 |
| **ステップ実行** | LAB-STEP: 1ステップごとの結果確認 |
| **パラメータ操作** | LAB-PARAM: スライダー操作で即座にグラフ更新 |

#### 1.5 教えるつもり学習（Protégé Effect）

[Chase et al. 2009]

「後で他人に教える」という状況設定により、**仮想的な予測誤差を先取り**させる。脳は「質問に答えられないかもしれない」という将来の誤差を予期し、情報のPrecision（解像度）を高めてサンプリングする。

| 要素 | Isidoricaでの実装（将来） |
| :--- | :--- |
| **説明タスク** | Quiz: 「この概念を友人に説明するなら？」形式 |
| **コミュニティ** | 将来: コメント機能での質問回答、編集への参加 |

> **参照**: Laboratory設計書の「予測の明示化」「最適な予測誤差の設計原則」を参照。

## 2. 認知負荷の最適化

**理論基盤**: 認知負荷理論（2.1）、マルチメディア学習の認知理論（2.7）、専門性逆転効果（2.4）

### 設計原則

ワーキングメモリには容量制限があるため、**外在的負荷を最小化**し、**学習関連負荷に資源を集中**させる。

- 視覚と言語の二重チャンネルを活用
- 対応する言葉と絵は近く（同時）に提示（近接性原理）
- 無関係な素材は排除（一貫性原理）
- ユーザーの熟達度に応じて足場かけを調整

> **参照**: USER_BEHAVIOR_MODEL.md セクション3.4における行動予測への適用

### 教育実践フレームワーク

#### 2.1 マルチメディア設計原則

[Mayer 2005]

| 原理 | 内容 | Isidoricaでの実装 |
| :--- | :--- | :--- |
| マルチメディア原理 | 言葉だけより、言葉と絵の方がよく学べる | テキストとLaboratory（シミュレーション）の統合 |
| 近接性原理 | 対応する言葉と絵は近く（同時）に提示 | 解説の直下に実験を配置 |
| 一貫性原理 | 無関係な素材は排除する | 装飾的要素を排除、機能的デザイン |

#### 2.2 例題活用効果（Worked Example Effect）

[Sweller & Cooper 1985]

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **完全例題** | 初心者向け: 解法を全てステップ表示 |
| **穴あき例題** | 中級者向け: 一部のステップを空欄に |
| **問題解決** | 上級者向け: 例題なし、自力で解く |

#### 2.3 段階的スキャフォールディング

[Wood, Bruner & Ross 1976]

ZPD内で学習者が課題を達成できるよう、一時的な支援（ヒント、ガイド、モデル提示）を行い、能力向上に伴って徐々に取り除く（Fading）。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **段階的支援** | 最初は詳細なガイド付きLaboratory、次はパラメータ操作のみ、最後は自由記述 |
| **動的なヒント** | 詰まった時だけ表示されるコンテキスト依存のヒント |
| **フェーディング** | 習熟度に応じて支援を減少 |

#### 2.4 Reader Adaptation（専門性逆転効果への対応）

[Kalyuga et al. 2003]

初心者向けの教育設計が、専門家には逆効果になる現象に対応する。

| モード | 想定熟達度 | 設計方針 |
|-------|----------|---------|
| **Explorer** | 初心者 | 最大限の足場かけ、詳細な説明、ステップバイステップ誘導 |
| **Learner** | 中級者 | 中程度の足場かけ、必要に応じたヒント |
| **Scholar** | 上級者 | 最小限の説明、詳細は折りたたみ、自己主導の探究 |

> **参照**: USER_BEHAVIOR_MODEL.md セクション3.8における行動予測への適用

## 3. 内発的動機づけの設計

**理論基盤**: 自己決定理論（3.1）、達成目標理論（3.5）、興味の発達モデル（3.4）

### 設計原則

内発的動機づけは、**自律性・有能感・関係性**の3つの基本的心理欲求が満たされた時に最大化される。

- **Autonomy**: 従来の教科書は「これを覚えろ」という強制（Autonomy阻害）が多い。本モデルでは、Laboratoryでの自由な探索、学習パスの選択自由度、自分のペースで進められるUIを通じて自律性を担保する。
- **Competence**: 単なるスコア表示や正誤判定ではなく、**「あなたの操作がどう成功につながったか」を示す情報的フィードバック（Informational Feedback）**を行う。統制的フィードバック（「よくできました、システムに従いましたね」）は避ける。
- **Relatedness**: 将来的なコミュニティ機能（コメント、議論、共同編集）を通じて、学習者が「知のコミュニティ」に所属している感覚を醸成する。

> **参照**: USER_BEHAVIOR_MODEL.md セクション7.1における行動予測への適用

### 教育実践フレームワーク

#### 3.1 自律性支援的指導（Autonomy-Supportive Teaching）

[Reeve 2006]

教師が学習者の選択を尊重し、統制的な指導を避けることで内発的動機づけを促進する。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **選択の提供** | 学習パスの選択自由度、記事の読み順を強制しない |
| **根拠の説明** | 「なぜこの概念を学ぶのか」を記事冒頭で明示 |
| **圧力の排除** | 期限やスコアによる外発的プレッシャーを避ける |

#### 3.2 習熟学習（Mastery Learning）

[Bloom 1968]

「成績」ではなく「習熟」を基準とし、全ての学習者が十分な時間をかければ習得可能という前提に立つ。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **習熟基準の明確化** | Quiz: 何ができれば「分かった」と言えるか明示 |
| **再試行の許容** | 正解するまで何度でも挑戦可能 |
| **進度の個別化** | 学習者のペースに合わせた進行 |

#### 3.3 フロー理論と最適チャレンジ（Flow & Optimal Challenge）

[Csikszentmihalyi 1990]

スキルと挑戦のバランスが取れた時、学習者は「フロー」と呼ばれる没入状態に入る。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **適切な難易度** | Laboratory: 段階的に複雑化（Goldilocks Zone） |
| **即時フィードバック** | 操作結果を即座に視覚化 |
| **明確な目標** | 各Laboratory、Quizの目的を冒頭で提示 |

> **参照**: Laboratory設計書の「最適な予測誤差の設計原則」を参照。

#### 3.4 協調学習（Cooperative Learning）

[Johnson & Johnson 1989]

相互依存的な課題を通じて、学習者同士が協力して目標を達成する。

| 設計原則 | Isidoricaでの実装（将来） |
| :--- | :--- |
| **相互依存性** | 共同編集、議論フォーラム |
| **個人の責任** | 各自の貢献が可視化される仕組み |
| **対面的相互作用** | コメント機能、質問回答 |

#### 3.5 探究学習（Inquiry-Based Learning）

[Bruner 1961]

教師が答えを提示するのではなく、学習者自身が問いを立て、探究を通じて発見する。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **問いの提示** | 記事冒頭で「なぜ？」を問いかける |
| **自由探索** | Laboratory: 自由探索モード、パラメータ操作 |
| **発見の促進** | 生産的失敗：解説前に試行錯誤させる |

#### 3.6 興味の発達支援

[Hidi & Renninger 2006]

Isidoricaの課題は、**状況興味を個人興味に発達させる**ことである。

| Phase | 興味の段階 | 設計方針 |
|-------|----------|---------|
| **Phase 1** | 誘発的状況興味 | 予測誤差、驚き、パズル的要素で状況興味を誘発 |
| **Phase 2-4** | 維持的状況興味 | 操作、発見、Aha!体験で状況興味を維持 |
| **Phase 5** | 萌芽的個人興味 | 文脈と意義の明示で個人興味への萌芽を促進 |
| **Agora** | 発達した個人興味 | 知識の共有と議論により、再関与の機会を提供 |

> **参照**: USER_BEHAVIOR_MODEL.md セクション7.6における行動予測への適用

## 4. 感情と学習体験

**理論基盤**: 統制-価値理論（3.2）、認識的感情（7.3）、フロー理論（3.3）

### 設計原則

学習場面で生じる感情は、**統制（Control）**と**価値（Value）**の2つの評価の組み合わせから予測される。

- **Phase 1（驚き→好奇心への橋渡し）**: 単に予測を裏切る（驚き）だけでは、統制感が欠如し「不安」や「無力感」に繋がるリスクがある。「驚き」と共に**「解決のためのハンドル（操作手段）」**を即座に提示することで、感情を「好奇心（High Control / High Value）」へと誘導する。
  - 例: 「あれ？おかしいな（驚き）」→「ここのスライダーを動かしてみよう（統制感の担保）」
- **Phase 4（フローの維持）**: 「退屈」は課題が簡単すぎる（価値低）、「不安」は課題が難しすぎる（統制低）場合に生じる。スキルと難易度のマッチングにより**享楽（Enjoyment）**を維持する。
- **怒りの防止（UI/UX）**: 学習者が「正しい操作をしている」と思っているのに期待通り動かない場合は「怒り」になる。LaboratoryのUI/UXには高い応答性と透明性が求められる。

> **参照**: USER_BEHAVIOR_MODEL.md セクション8における行動予測への適用

### 教育実践フレームワーク

#### 4.1 帰属再訓練（Attribution Retraining）

[Weiner 1985]

学習者が失敗を「自分の能力がない」からではなく「努力や方略が不足していた」と帰属するよう促す。これにより統制感（Control）を維持する。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **能力否定の回避** | Quiz失敗時：「間違い」ではなく「もう一度考えてみよう」 |
| **方略の提示** | 「ヒントを見る」「前の章を復習する」オプションの提供 |
| **努力の価値化** | 「正解を導くまでに3回試行しました」など過程の可視化 |

#### 4.2 生産的混乱（Productive Confusion）

[D'Mello & Graesser 2012]

適度な混乱は深い学習を促進する。ただし、混乱が長引くと不安・欲求不満に変化するため、適切なタイミングで解決の手がかりを提供する必要がある。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **意図的な混乱の誘発** | Phase 1: 予測を裏切る結果を見せる |
| **タイムアウトガード** | 一定時間解決しない場合、ヒントを自動表示 |
| **段階的ヒント** | 最初は曖昧なヒント、必要に応じて具体的に |

| ゾーン | 特徴 | 設計対応 |
|-------|------|---------|
| **混乱なし（退屈）** | 既知の内容、挑戦なし | 難易度を上げる |
| **生産的混乱** | 解決可能な不整合 | 維持、必要に応じヒント |
| **絶望的混乱** | 解決不可能な不整合 | 足場かけ強化 |

> **参照**: 1.2節の「生産的失敗（Productive Failure）」との連携。

#### 4.3 価値介入（Utility-Value Intervention）

[Hulleman & Harackiewicz 2009]

学習内容と学習者の個人的目標・興味との関連性を明示することで、知覚される価値（Value）を高める。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **実用性の明示** | 記事冒頭：「この概念は○○に使える」 |
| **接続の促進** | 「あなたの分野ではどう応用できるか？」問いかけ |
| **多様な応用例** | 遠隔転移（Far Transfer）の例を複数提示 |

#### 4.4 感情調整指導（Emotion Regulation Instruction）

[Pekrun & Perry 2014]

学習者自身が自分の感情をモニタリングし、適切に制御するスキルを育成する。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **感情の自己認識** | Quiz: 「今どのくらい自信がある？」（JOLチェック） |
| **混乱の正常化** | 「混乱は学習の一部です」というメッセージ |
| **休憩の推奨** | 長時間利用時：「休憩を取りましょう」の通知 |

#### 4.5 心理的安全性と失敗許容（Psychological Safety）

[Edmondson 1999]

失敗がペナルティなく許容される環境を作ることで、統制感の喪失を防ぐ。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **ペナルティなし** | 何度でも再試行可能、スコアによる序列化なし |
| **失敗の再フレーミング** | 「間違い」を「発見」として表現 |
| **匿名性の担保** | 学習履歴の非公開（他者との比較を排除） |

## 5. 適応的な難易度調整

**理論基盤**: 最近接発達領域（ZPD）（7.1）、専門性逆転効果（2.4）、フロー理論（3.3）

### 設計原則

最適な学習は、現在の能力をわずかに超えた「最近接発達領域（ZPD）」で起こる。

- スキルと挑戦のバランスを維持
- ユーザーの習熟度に応じて動的に調整
- 適度な「生産的混乱」を維持しつつ、「絶望的混乱」を防止

### 教育実践フレームワーク

#### 5.1 概念変容モデル（Conceptual Change Model）

[Posner et al. 1982]

学習者が持つ「日常的概念（誤概念）」を「科学的概念」へと変容させるための条件を提示した。

1. **不満（Dissatisfaction）**: 既存の概念では現象を説明できないと痛感する。
2. **理解可能性（Intelligibility）**: 新しい概念（科学的概念）の意味が理解できる。
3. **もっともらしさ（Plausibility）**: 新しい概念が正しいように思える。
4. **有効性（Fruitfulness）**: 新しい概念を使うと別の問題も解決できる。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **不満の醸成** | Phase 1: 予測を裏切る実験結果により、素朴概念への不満を生む |
| **理解と受容** | Phase 2: 直観と整合的な定義を与え、もっともらしさを担保する |
| **有効性の実感** | Phase 3-4: その定義を使うと複雑な現象も説明できることを示す |

#### 5.2 類推による架橋（Bridging Analogies）

[Clement 1993]

直観的に理解しやすい「アンカー（Anchor）」と、理解困難な「ターゲット（Target）」の間を、中間的な事例（ブリッジ）で繋ぐことで、日常的概念を科学的モデルへと拡張する。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **アンカーの提示** | 抽象的概念の導入に、誰もが納得する日常的事例を使う（例：作用反作用の「バネ」） |
| **ブリッジの構築** | Laboratoryでアンカーからターゲットへ徐々にパラメータを変形させる |

#### 5.3 認知的徒弟制（Cognitive Apprenticeship）

[Collins et al. 1989]

専門家の思考プロセス（科学的概念の運用方法）を可視化（Modelling）し、学習者に模倣させ、コーチングし、最終的に自立させる。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **思考の可視化** | Phase 2-3: 「なぜこの定義を使うのか」「どう式変形するか」の内言（Inner Speech）を記述 |
| **比較と省察** | ユーザーの解答プロセス（Laboratory操作ログ）と、理想的なモデルを並べて比較 |

> **参照**: 2.3節の「段階的スキャフォールディング」と連携。

## 8. 非競争的・包括的設計

**理論基盤**: 教育的公正、社会的比較理論（5.2）、認知的倹約家（5.1）

### 設計原則

競争ではなく**熟達**に焦点を当て、全ての学習者をエンパワーする。

- **認知的倹約家を前提とした設計**: ユーザーは「怠惰」なのではなく、認知資源を効率的に使おうとしている
- **深い学習への誘引**: 強制ではなく、報酬（Aha!体験、美的報酬）で自発的な思考を誘発
- ランキング、偏差値、相対評価は使用しない
- ユーザーの「最小努力」を認知的倹約として尊重しつつ、深い学習が必要な場面でのみ介入

> **参照**: USER_BEHAVIOR_MODEL.md セクション6.1における行動予測への適用。これはIsidorica設計の中核的前提である。

### 教育実践フレームワーク

#### 8.1 非競争的社会的証明

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **競争要素の排除** | ランキング、偏差値、相対評価は使用しない |
| **共感的社会的証明** | 「多くの人がこの部分でつまずきます」 |
| **匿名性の保護** | 個人の学習履歴を他者に開示しない |
| **熟達目標の促進** | 他者との比較ではなく、自己の成長に焦点 |

#### 8.2 認知的倹約家への配慮

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **効率的な経路提供** | 情報の香りを最適化、深い学習への経路を明示 |
| **誘引による介入** | 強制ではなく、報酬（Aha!体験、美的報酬）で自発的な思考を誘発 |
| **満足化への対応** | 深い学習が必要な場面でのみ軽い介入 |

#### 8.3 批判的教育学（Critical Pedagogy）

[Freire 1970]

学習者を「空っぽの容器（Banking Model）」として扱わず、その**生きられた経験**と知識を接続する。

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **経験の尊重** | Phase 1: 学習者の素朴概念（先入見）を出発点として活用 |
| **対話的関係** | 一方的な知識伝達ではなく、Laboratoryでの双方向的探索 |

> **連携**: ブルデューの「合理的教育学」とも共鳴し、知識の民主化に貢献する。

#### 8.4 3層の対話モデル（擬似的CoP）

「個人学習ツールのIsidoricaでは、LPP（コミュニティへの参加）が成立しないのではないか（孤独な正統的周辺参加）」という批判に対し、本モデルは**「対話」の概念を拡張**することで応答する。

| 層 | 対話の相手 | 説明 | Isidoricaでの実装 |
|----|----------|-----|------------------|
| **1. 垂直的対話 (Vertical Dialogue)** | **著者・先人** | テキストやLaboratory（物象化された知恵）を通じた、専門家の思考の追体験。 | 記事、Laboratory |
| **2. 間接的対話 (Indirect Dialogue)** | **群衆 (The Crowd)** | 統計データを通じた他者の気配（Social Presence）の共有。「自分は一人ではない」という感覚。**ただし、競争を煽る「評価的指標（ランキング等）」は排除し、「共感・連帯的指標（つまずきの共有等）」に限定する。** | 思考の痕跡、みんなのハイライト（将来） |
| **3. 水平的対話 (Horizontal Dialogue)** | **仲間 (Peers)** | 「問い」や「気づき」を共有する場。質の低いQ&Aではなく、探究のための対話。 | **Discussions (Agora)** |

> **設計方針**: Isidoricaは物理的な「教室」を持たないが、これら3つの対話を重ねることで、ユーザーを**「孤独だが、孤立していない（Solitary but not Isolated）」**状態に置く。特に「間接的対話」においては、他者との比較による劣等感（遂行目標の喚起）を防ぐため、**「能力の競争」ではなく「経験の共有」**をデザインの主眼に置く。

## 11. 前提知識と学習パス設計

**理論基盤**: 動的意味論（9.1）、科学的概念と日常的概念（7.1）

### 設計原則

記事間の前提関係を**動的意味論**の枠組みで形式化する。

- **文脈（Context）** = ユーザーが学習済みの概念の集合 $K$
- **文脈更新（Context Update）** = ユーザーが記事を読んで概念を獲得する
- **前提（Presupposition）** = 記事の前提知識（`requires` フィールド）
- **アコモデーション** = 前提が満たされていなくても概要を理解できる自己完結的説明

> **参照**: USER_BEHAVIOR_MODEL.md セクション5.3における行動予測への適用

### 教育実践フレームワーク

#### 11.1 前提知識システム

| 動的意味論の概念 | Isidoricaでの応用 |
| :--- | :--- |
| **文脈（Context）** | ユーザーが学習済みの概念の集合 $K$ |
| **文脈更新（Context Update）** | ユーザーが記事を読んで概念を獲得する |
| **前提（Presupposition）** | 記事の前提知識（`requires` フィールド） |
| **適正性条件（Felicity Condition）** | 前提記事を読まずに高度な記事にアクセスした場合 |

## 12. テキスト・ナビゲーション設計（心理言語学）

**理論基盤**: メンタルレキシコン（10.1）、意味的プライミング（10.3）、文処理（10.4）、二言語処理（10.5）

### 設計原則

読者がどのように語を認識し、意味を処理するかを踏まえ、**記事テキスト・ナビゲーション・辞書機能**を設計する。

- メンタルレキシコンの構造（音韻・正書法・統語・意味・語用）を反映した辞書スキーマ
- 活性化の広がり（Spreading Activation）に基づく関連語ナビゲーション
- 増分的処理を妨げない明確な文構造
- 二言語話者の概念レベル共有を活用した多言語設計

> **注**: 心理言語学は辞書スキーマ設計だけでなく、記事テキストの可読性、関連項目のナビゲーション、多言語対応にも適用される。

### 教育実践フレームワーク

#### 12.1 辞書スキーマとメンタルレキシコン

| メンタルレキシコンの構成要素 | Isidoricaスキーマのフィールド |
| :--- | :--- |
| 音韻表象 | `form.pronunciation` |
| 正書法表象 | `form.orthography` |
| 統語情報 | `syntax.arg_structure` |
| 意味情報 | `qualia`, `ecology`, `relations` |
| 語用情報 | `pragmatics.register`, `pragmatics.collocation` |

#### 12.2 プライミング効果とナビゲーション

[Meyer & Schvaneveldt 1971]

意味的に関連する語を近くに配置することで、語彙アクセスと学習効率が向上する。

| 設計原則 | Isidoricaでの実装 |

#### 12.3 テキスト可読性（文処理）

## 13. 認識的信念と成長マインドセット

**理論基盤**: 認識的信念（7.2）、マインドセット理論

### 設計原則

学習者が持つ**「知識とは何か」「学習はどう起こるか」についての信念**が、学習行動と成果に影響を与える。

- 「知識は単純で固定的」という素朴信念への対処
- 「学習は即座に起こるか、全く起きない」という信念の変容
- 成長マインドセット（能力は発達可能）の促進

> **参照**: USER_BEHAVIOR_MODEL.md セクション7.4における行動予測への適用

### 教育実践フレームワーク

#### 13.1 素朴な認識的信念への対処

[Schommer 1990]

| 素朴な信念 | リスク | Isidoricaでの対処 |
| :--- | :--- | :--- |
| **知識は単純** | 複雑な説明を「不適切な教材」と判断 | Phase 3で複雑さの意義を説明 |
| **知識は確実** | 暫定的な科学知識を受け入れられない | 「現在の理解では」という表現 |
| **学習は即座** | 時間をかける学習を「自分に合わない」と回避 | 「習熟には時間がかかる」の正常化 |
| **能力は固定** | 失敗を能力の欠如と帰属し離脱 | 成長マインドセットの促進 |

#### 13.2 信念の変容支援

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **複雑性の段階的開示** | Phase 1-2はシンプルに、Phase 3以降で複雑性を明示 |
| **暫定性の明示** | 「〜と考えられている」「現在の研究では」 |
| **努力の価値化** | 「3回目の試行で正解しました」など過程の可視化 |
| **成長の証拠** | 過去の自分との比較（他者比較ではなく） |

#### 13.3 成長マインドセットの促進

[Dweck 2006]

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **能力ではなくプロセスを称賛** | 「良い戦略ですね」「努力が実りました」 |
| **「まだ」の力** | 「まだ理解できていない」（not yet） |
| **挑戦の価値化** | 「難しい問題は脳を鍛えます」 |

## 14. 時間割引と即時性のバランス

**理論基盤**: 時間割引と現在バイアス（4.3）、双曲割引

### 設計原則

人間は**将来の報酬を割り引いて評価**し、即時の報酬を過大評価する傾向がある。学習は「今の苦痛」と「将来の利益」のトレードオフを含むため、この傾向への対処が必要。

- 即時的な報酬（Aha!体験、視覚的フィードバック）を設計に埋め込む
- 遅延報酬の価値を可視化・具体化
- コミットメントデバイスの提供

> **参照**: USER_BEHAVIOR_MODEL.md セクション4.3における行動予測への適用

### 教育実践フレームワーク

#### 14.1 即時報酬の埋め込み

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **即時フィードバック** | Laboratory操作→即座にグラフ更新 |
| **Aha!体験の設計** | Phase 1-3での予測誤差→解決の快感 |
| **進捗の可視化** | 学習状況の視覚的表示（即時の達成感） |
| **マイクロ報酬** | 小さな成功の積み重ね |

#### 14.2 遅延報酬の価値化

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **将来価値の具体化** | 「この概念が分かると○○ができるようになる」 |
| **ロードマップ提示** | 学習パスの全体像と現在位置 |
| **成功事例の共有** | 学習者の成長ストーリー（将来）（非競争的に） |

#### 14.3 先延ばし対策

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **タスクの細分化** | 大きな目標を小さなステップに |
| **開始障壁の低減** | 「5分だけ試す」オプション |
| **中断からの復帰** | 前回の続きから再開しやすい設計 |

## 15. 生態学的合理性と環境設計

**理論基盤**: 生態学的合理性（2.5）、適応的ツールボックス

### 設計原則

人間の認知は特定の環境構造に適応して進化した。**シンプルなヒューリスティック**が機能する環境を設計することで、認知負荷を下げつつ適切な判断を促進できる。

- 複雑な意思決定を簡素化する環境構造
- 情報過多を防ぐキュレーション
- 自然な認知様式に沿ったインターフェース

### 教育実践フレームワーク

#### 15.1 ヒューリスティックが機能する環境設計

[Gigerenzer & Gaissmaier 2011]

| ヒューリスティック | 機能する条件 | Isidoricaでの応用 |
| :--- | :--- | :--- |
| **再認ヒューリスティック** | 認識できるものを選ぶ | よく見る概念を重要と判断 → 頻出概念の視認性向上 |
| **Take-the-best** | 最も有効な手がかりで判断 | 最も重要な情報を最初に提示 |
| **デフォルト** | 変更しなければ現状維持 | 推奨学習パスをデフォルトに |

#### 15.2 情報環境の構造化

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **情報の段階的開示** | 最初は概要、必要に応じて詳細へ |
| **カテゴリ構造の明確化** | 概念間の関係を視覚的に表現 |
| **選択肢の限定** | 「次のおすすめ」を3つ程度に絞る |
| **比較の容易化** | 類似概念を並列表示 |

#### 15.3 認知的ニッチ構築

| 設計原則 | Isidoricaでの実装 |
| :--- | :--- |
| **外部記憶の活用** | ブックマーク、学習履歴の保存 |
| **認知のオフロード** | 複雑な計算はLaboratoryに委託 |
| **環境への情報埋め込み** | 関連情報のリンク配置 |
