# コンテンツポリシー (Content Policy)

本ドキュメントでは、Isidoricaにおけるコンテンツの品質基準と禁止事項を定義する。

> **関連文書**:
> - [GOVERNANCE.md](../library/GOVERNANCE.md): 執筆・レビュー・出典管理のプロセス
> - [DISCUSSIONS_DESIGN.md](../library/DISCUSSIONS_DESIGN.md): Agoraにおける議論のポリシー

---

## 0. 本ポリシーの適用範囲

本ポリシーは、Isidorica上で公開される**記事、ドキュメント、コード等の「コンテンツ」**に適用される。
コミュニティ参加者の「対人行動（コミュニケーション）」については、[CODE_OF_CONDUCT.md](../library/CODE_OF_CONDUCT.md) が適用される。

Isidoricaのコンテンツには、以下の2つの基準が求められる。

1.  **倫理的基準 (Ethical Standards)**: 他者を不当に傷つけないこと（§1）
2.  **認識的基準 (Epistemic Standards)**: 知識としての正確性と信頼性を担保すること（§2, §3）

---

## 1. 倫理的基準：禁止コンテンツ

### 1.1 明示的禁止事項

| 禁止事項 | 説明 |
| :--- | :--- |
| **広告的内容** | 特定の製品・サービスを過度に推奨する記述 |
| **独自研究** | 検証可能な出典のない独自の結論や因果関係の断定 |
| **明示的差別語** | Slurs（侮蔑語）、ヘイトスピーチ（CoC違反内容を含む） |

### 1.2 暗黙的な差別表現 (Implicit Discrimination)

言語哲学（語用論）に基づき、以下の暗黙的な差別表現も制限対象とする。

| 現象 | 説明 | 例 |
| :--- | :--- | :--- |
| **Dogwhistles（犬笛）** | 特定の集団にのみ差別的意図が伝わる表現 | 「都市部の若者」が特定人種を暗示する場合 |
| **Toxic Implicature（有害な含意）** | 含意によって偏見を強化する表現 | 「X人にしては勤勉だ」（X人は怠惰という前提） |
| **Illocutionary Harm（発語内的害）** | 発話行為そのものが害を構成する表現 | カテゴリ化による抑圧的断言 |
| **Microaggression（マイクロアグレッション）** | 日常的な、しばしば無意識の軽蔑的メッセージ | 「日本語上手ですね」（外見で判断） |

> **判定に関する注記**:
> 暗黙的差別は文脈依存性が高く、機械的な判定が困難である。
> レビューにおいて判断が割れた場合は、直ちに禁止とせず、**[Agora](../library/DISCUSSIONS_DESIGN.md) にてコミュニティの合意形成（熟議）を行う**ものとする。

## 2. AI使用ポリシー

### 2.1 基本方針

| 項目 | 決定 |
| :--- | :--- |
| **AI執筆の使用** | 禁止も推奨もしない |
| **AI翻訳の使用** | 禁止も推奨もしない |
| **AI使用の開示義務** | なし（判別困難なため） |
| **出典の人間確認** | **必須** |

### 2.2 核心的なリスク：出典捏造（ハルシネーション）

AIによる記事生成の最大のリスクは、**存在しない出典を自信満々に提示する「ハルシネーション」**である。

| リスク | 説明 |
| :--- | :--- |
| **出典の捏造** | 実在しない書籍・論文を引用する |
| **出典の誤引用** | 実在する出典だが、内容と異なる主張を帰属させる |
| **出典の過剰信頼** | 信頼性の低い情報源を権威あるものとして引用する |

### 2.3 AIガイドライン（AI機能設計向け）

Isidorica内で使用されるAI機能（将来のAIファシリテーター等）は、以下のガイドラインに従う。

| ガイドライン | 説明 |
| :--- | :--- |
| **never_reinforce_user_bias** | ユーザーのバイアスを強化しない |
| **flag_presuppositions** | 問題のある前提を指摘する |
| **provide_counternarratives** | 対抗的な視点を提示する |

> **理論的基盤**: 本ポリシーのAI使用に関する方針は、[UNESCO AI倫理勧告](https://www.unesco.org/en/artificial-intelligence/recommendation-ethics)の透明性・説明責任原則に沿う。

## 3. 認識的基準：出典の信頼性階層

### 3.1 Tier分類

| Tier | 種別 | 例 | 許容度 |
| :--- | :--- | :--- | :--- |
| **Tier 1（推奨）** | 査読付き学術出版物 | 学術論文、学術書、大学出版局の書籍 | ✅ 推奨 |
| **Tier 2（許容）** | 権威ある機関の公式情報 | 政府機関、国際機関（WHO、UN等）、公式統計、法令・判例 | ✅ 許容 |
| **Tier 3（許容）** | 専門家による信頼性の高い情報 | 教科書、公式ドキュメント（MDN、RFC等）、信頼性の高い報道機関 | ✅ 許容 |
| **Tier 4（条件付き）** | 専門家のプレプリント・解説 | arXiv、bioRxiv、専門家ブログ、講義ノート | ⚠️ 条件付き |
| **Tier 5（限定的）** | 一般的なWebソース | 個人ブログ、SNS、Wikipedia | ❌ 原則禁止 |

### 3.2 Tier別の使用条件

#### Tier 1: 査読付き学術出版物（推奨）

| 条件 | 説明 |
| :--- | :--- |
| **使用条件** | 常に許容、推奨 |
| **特記事項** | 複数の出典がある場合、査読付きを優先 |

#### Tier 2: 権威ある機関の公式情報（許容）

| 条件 | 説明 |
| :--- | :--- |
| **使用条件** | 常に許容 |
| **適用分野** | 法律、政策、統計、医療ガイドライン等 |
| **例** | 厚生労働省、FDA、WHO、最高裁判例 |

#### Tier 3: 専門家による信頼性の高い情報（許容）

| 条件 | 説明 |
| :--- | :--- |
| **使用条件** | 常に許容 |
| **適用分野** | 技術ドキュメント、教科書、ニュース報道 |
| **例** | MDN Web Docs、RFC、O'Reilly書籍、Reuters/AFP等 |

#### Tier 4: 専門家のプレプリント・解説（条件付き）

| 条件 | 説明 |
| :--- | :--- |
| **使用条件** | Tier 1-3の出典がない場合のみ |
| **追加要件** | 「プレプリントであり査読を経ていない」ことを記事内で明記 |
| **適用分野** | 最新研究、専門家の解説 |
| **例** | arXiv、bioRxiv、著名研究者のブログ |

> **注意**: プレプリントは後に撤回される可能性があるため、定期的見直し対象とする。

#### Tier 5: 一般的なWebソース（原則禁止・例外あり）

| 条件 | 説明 |
| :--- | :--- |
| **使用条件** | 原則禁止。「事実の立証（Authority）」として用いてはならない。 |
| **例外 (1)** | **言及対象**: 「SNSでの反応」そのものを論じる際の一次資料として。 |
| **例外 (2)** | **対抗的語り (Counter-Narratives)**: ある集団が特定の経験をしていること自体を示す**「証拠（Evidence）」**として用いる場合。 |
| **記述ルール** | 「Aによれば事実はXである」ではなく、「AはXという経験を語っている」と記述する。 |
| **例** | Wikipedia、個人ブログ、SNS投稿 |

### 3.3 分野別の考慮事項

| 分野 | 推奨出典 | 備考 |
| :--- | :--- | :--- |
| **哲学・人文学** | 原典、学術書、学術論文 | 原典の翻訳版を使用する場合は翻訳者・出版社を明記 |
| **社会科学** | 査読付き論文、公的統計 | 定性的研究において当事者の語り（Tier 5）を一次資料として扱うことは許容される |
| **自然科学** | 査読付き論文、教科書 | プレプリントは注意付きで許容 |
| **法律** | 法令、判例、政府解説 | 学術論文は補足として |
| **技術（IT）** | 公式ドキュメント、RFC、仕様書 | 個人ブログは原則禁止、ただし公式ブログは許容 |
| **医療・健康** | 査読付き論文、公的機関のガイドライン | **Tier 4-5は禁止**（生命に関わるため例外なし） |

### 3.4 出典の時系列的妥当性

出典の信頼性は、その**発行時期**にも依存する。

| 分野 | 推奨有効期間 | 備考 |
| :--- | :--- | :--- |
| **技術（IT）** | 5年以内 | 技術進化が速いため。古い出典は「版」を明記。 |
| **自然科学** | 10年以内 | 定説・教科書はこの限りではない。 |
| **社会科学** | 10-20年 | 統計データは最新を優先。 |
| **哲学・人文学** | 制限なし | 古典は時間に依存しない。 |
| **法律** | 現行法 | 改正後は速やかに更新。 |

> **注意**: 古い出典が不適切という意味ではない。ただし、「この出典はXXXX年のものであり、現在は状況が異なる可能性がある」と明記することが推奨される。

## 4. 違反時の対応

| 違反 | 対応 |
| :--- | :--- |
| **過失による誤引用** | 記事の修正・出典の削除 |
| **悪意のある捏造** | 記事の削除、GitHubアカウントのブロック |
| **有害言語の使用** | 記事の修正、繰り返しの場合はブロック |

> **注意**: 「悪意のある捏造」の認定は、明らかに存在しない出典を複数回・組織的に使用した場合に限定する。単発の誤りは過失として扱う。

### 4.1 異議申立てプロセス

ブロック決定に対する異議申立ては、[GOVERNANCE.md](../library/GOVERNANCE.md) の「Dispute Resolution」セクションに従う。

### 4.2 復帰の原則

Isidoricaは「教育プラットフォーム」であり、ブロックは永久追放ではなく、**行動改善の機会**として位置づける。ただし、コミュニティの安全が最優先される。復帰の具体的なプロセスは[GOVERNANCE.md](../library/GOVERNANCE.md)に定める。
