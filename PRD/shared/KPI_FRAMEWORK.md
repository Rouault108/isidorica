# KPIフレームワーク (Key Performance Indicators Framework)

本ドキュメントでは、Isidoricaの成功を測定するための指標体系を定義する。

**ステータス**: 確定
**最終更新**: 2026-01-08

---

## 1. 基本方針

### 1.1 Isidoricaの特殊性

Isidoricaは**「成長」より「信頼」を優先する公共図書館**である。従来のWebサービスKPI（MAU、コンバージョン率、広告収入など）とは異なるアプローチを採用する。

| 従来のWebサービス | Isidorica |
| :--- | :--- |
| ユーザー数の最大化 | 学習効果の最大化 |
| エンゲージメントの最大化 | 理解の深化 |
| 収益の最大化 | 社会的価値の最大化 |
| バイラル拡散 | 信頼性の構築 |

### 1.2 成功の定義

Isidoricaが成功した状態とは：

1. **学習者が実際に理解できている**（学習効果）
2. **信頼できる情報源として認知されている**（信頼性）
3. **すべての人が公正にアクセスできている**（公正性）
4. **社会的価値を提供している**（公共知としての役割）
5. **長期的に運営が維持されている**（持続可能性）

### 1.3 哲学との整合性

| 哲学・原則 | KPIへの反映 |
| :--- | :--- |
| **信頼性 (Trustworthiness)** | 教育機関採用数、誤り報告率 |
| **学習体験 (Learning Experience)** | 読者評価（自発的フィードバック） |
| **教育的公正 (Educational Equity)** | アクセシビリティスコア、多言語到達率 |
| **公共知としての役割** | 教育機関での採用 |
| **持続可能性** | 運営コスト、コントリビューター数 |

## 2. 成功の因果モデル (Logic Model)

KPIは単独で存在するのではなく、因果関係のネットワークを形成する。

### 2.1 因果連鎖

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                成功の因果モデル                                                        │
└─────────────────────────────────────────────────────────────────────────────┘

  [INPUT]                [ACTIVITIES]               [OUTPUT]                [OUTCOME]                   [IMPACT]
  投入資源                      活動                     産出物                    成果                         影響
    │                         │                       │                       │                          │
    ▼                         ▼                       ▼                       ▼                          ▼
┌─────────┐      ┌──────────┐      ┌───────────┐        ┌───────────┐        ┌───────────┐
│・資金         │ ──→ │・記事執筆       │ ──→ │・公開記事数       │ ──→  │・学習効果         │ ──→   │・知識の社会還元     │
│・人的資源     │       │・レビュー       │       │・LP数           │        │・信頼獲得         │        │（北極星）         │
│・技術基盤      │      │・翻訳           │      │・翻訳記事数       │        │・公正アクセス      │        │                 │
└─────────┘      └──────────┘      └───────────┘        └───────────┘        └───────────┘
     │                                                                                                         │
     └──────────────────────────────────────────────────────────────────────┘
                                            持続可能性（フィードバックループ）
```

### 2.2 指標カテゴリ間の因果関係

| 先行指標 | 結果指標 | 因果仮説 |
| :--- | :--- | :--- |
| **持続可能性** → 信頼性 | コントリビューター数 → 専門家レビュー率 | 人的資源が品質を支える |
| **信頼性** → 学習効果 | 専門家レビュー率 → 読者評価 | 正確な情報が理解を促進する |
| **公正性** → 学習効果 | アクセシビリティ → 多様な学習者の成功 | バリアの除去が学習を可能にする |
| **学習効果** → 北極星 | 読者評価 → 教育機関採用 | 効果が実証されると採用される |

## 3. 北極星指標 (North Star Metric)

### 3.1 定義

> **知識の還元数** = Isidoricaのコンテンツが教育で引用・利用された累積回数

### 3.2 選択理由

| 理由 | 説明 |
| :--- | :--- |
| **哲学との整合性** | 「公共知としての役割」という使命と直結 |
| **操作不可能性** | 外部からの評価であり、自己評価では操作できない |
| **複合指標** | 学習効果、信頼性、社会的価値を全て反映 |

### 3.3 構成要素と測定方法

| 要素 | 操作的定義 | 測定方法 | 重み | 実現可能性 |
| :--- | :--- | :--- | :--- | :--- |
| **教育機関での採用** | 大学シラバス・高校教材での言及回数 | 手動調査、Google Alerts | 3 | △ 労力大、網羅性に限界 |
| **他プロジェクトでの利用** | Wikipedia、OSSドキュメントでの引用回数 | Search Console外部リンク | 1 | ○ 自動取得可能 |
| **AI/LLMでの参照** | 主要AIがIsidoricaを情報源として提示した回数 | 定期的サンプル調査（将来） | 1 | × 方法未確立、将来実装 |

**集計方法**: 加重合計 = Σ（各要素の件数 × 重み）

> **注**: 北極星指標は理念として重要だが、測定の網羅性と再現性に限界がある。特に「教育機関での採用」はリンクを伴わない引用（紙のシラバス等）を検出できない。現実的には、**Leading指標（§3.4）を日常的に追跡し、北極星は年次の参考値として扱う**。

### 3.4 Leading / Lagging 指標への分解

北極星は**Lagging指標**（結果指標）であり、直接的に改善行動を導きにくい。そのため**Leading指標**（先行指標）を設定し、日常的に追跡する。

| 指標タイプ | 指標 | 測定頻度 | 北極星への寄与 |
| :--- | :--- | :--- | :--- |
| **Lagging** | 知識の還元数（加重合計） | 年次 | — |
| **Leading** | 外部サイト被リンク数 | 月次 | 引用の先行指標 |
| **Leading** | 専門家レビュー済み記事率 | 月次 | 信頼性→採用の先行指標 |
| **Leading** | 読者評価 | 月次 | 学習効果→採用の先行指標 |

## 5. 信頼性指標

信頼性はIsidoricaの核心的価値であり、最も重要な指標カテゴリである。

### 5.1 Isidoricaの位置づけと指標設計

> 信頼性指標は、**教育現場での採用**、**読者からの評価**、**コンテンツの正確性**に焦点を当てる。

### 5.2 教育機関採用数

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 大学シラバス・高校教材・研修資料等でIsidoricaが参考資料として言及された回数 |
| **測定方法** | Google Alerts、シラバス検索、教育機関からの連絡、手動調査 |
| **頻度** | 年次 |
| **目標設定** | Phase 7公開後1年でベースライン測定 |
| **理論的根拠** | 教育機関での採用は、専門家による間接的な品質保証を意味する |

### 5.3 読者評価

学習効果・信頼性の両方を反映する指標。詳細は [§6.3 読者評価](#63-読者評価) を参照。

### 5.4 誤り報告率

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 事実に基づく誤り報告数 ÷ ページビュー |
| **測定方法** | GitHub Issues（`error-report`ラベル）、報告フォーム |
| **頻度** | 月次 |
| **目標** | 0.001%未満（PV10万あたり誤り報告1件未満） |
| **解釈** | 低いほど良い。ただし0は「誰も読んでいない」可能性も |

### 5.5 専門家レビュー率

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 専門家によるレビューを受けた記事の割合 |
| **計算式** | 専門家レビュー済み記事数 ÷ 公開記事数 |
| **測定方法** | GitHub PR承認履歴 + 専門家レジストリ照合 |
| **頻度** | 月次 |
| **目標（Phase 7 + 1年）** | 50%以上 |

> **実装詳細**: 専門家レジストリの管理については [GOVERNANCE.md §専門家レジストリ](../library/GOVERNANCE.md)、追跡ワークフローについては [CI_CD_PIPELINE.md §9.2](../engine/technical/CI_CD_PIPELINE.md) を参照。

## 6. 学習効果指標

### 6.1 哲学的立場

> **Isidoricaは、学習者のプライバシーを尊重し、入力品質で学習効果を保証する。**

| 原則 | 説明 |
| :--- | :--- |
| **信頼主義** | 学習効果は、**入力品質**（出典の信頼性、専門家レビュー、認知科学に基づくコンテンツ設計）によって間接的に保証する |
| **学習者主権** | 学習者自身が自分のデータを管理する。運営側が学習者の代わりに管理することはない |
| **自発的フィードバック** | 学習効果の評価は、学習者自身による**明示的・自発的**なフィードバックに依存する |

#### 根拠

公共図書館は、利用者がどの本で苦労したか、どの章を理解できなかったかを追跡しない。図書館の成功は、蔵書の質と利用者からの評価で測られる。Isidoricaも同様のアプローチを採用する。

> **参照**: [CORE_FOUNDATIONS.md §3](./CORE_FOUNDATIONS.md)（認知・行動基盤）、[PHILOSOPHY_DISCUSSION.md](./PHILOSOPHY_DISCUSSION.md)（プライバシー原則）

### 6.2 入力品質による間接的保証

学習効果を直接測定する代わりに、**学習効果を高める設計要素**の実装状況を確認する。

| 要素 | 保証方法 | 参照 |
| :--- | :--- | :--- |
| **認知科学に基づく設計** | LEARNING_FOUNDATIONS.mdに基づくコンテンツ設計 | [LEARNING_FOUNDATIONS.md](./LEARNING_FOUNDATIONS.md) |
| **予測誤差の喚起** | Laboratory設計パターンの適用 | [LABORATORY_DESIGN.md §2](../engine/LABORATORY_DESIGN.md) |
| **出典の信頼性** | 出典Tier分類、専門家レビュー | [GOVERNANCE.md §1](../library/GOVERNANCE.md) |
| **アクセシビリティ** | WCAG 2.2 AA準拠 | [公正性指標 §7](#7-公正性指標) |

### 6.3 読者評価

**唯一の学習効果関連KPI**として、読者からの自発的フィードバックを測定する。

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 記事末尾の「この記事は理解の助けになりましたか？」への肯定的回答率 |
| **計算式** | 肯定的回答数 ÷ 回答総数 |
| **測定方法** | 記事末尾の簡易フィードバックUI（2択：Yes/No） |
| **データ収集** | 匿名イベントのみ送信（記事ID + Yes/No）、ユーザー識別子なし |
| **頻度** | 月次 |
| **目標** | 80%以上 |

#### 設計上の注意

| 注意点 | 対応 |
| :--- | :--- |
| **自発的参加** | フィードバックは任意。回答しなくても読者体験に影響なし |
| **最小限のデータ** | 記事IDと回答（Yes/No）のみ。テキスト入力や追加情報は求めない |
| **回答者バイアス** | 回答者は満足/不満足が強い人に偏る可能性を認識。絶対値より傾向を重視 |

### 6.4 定性的フィードバック

数値化されないフィードバックも重視する。

| チャネル | 内容 | 運営側の対応 |
| :--- | :--- | :--- |
| **Agora（Discussions）** | 記事への質問、気づきの共有 | 傾向を把握し、記事改善に反映 |
| **GitHub Issues** | 具体的な改善提案、誤り報告 | 個別対応 |
| **お問い合わせフォーム** | 非GitHubユーザーからのフィードバック | 個別対応 |

### 6.5 学習者向け機能との区別

学習者**自身のため**の機能と、**運営側のKPI測定**は明確に区別する。

| 機能 | 目的 | データ保存 | サーバー送信 |
| :--- | :--- | :--- | :--- |
| **自己説明ノート** | 学習者自身の理解の確認 | LocalStorageのみ | ❌ なし |
| **Learning Path進捗** | 学習者自身の進捗管理 | LocalStorageのみ | ❌ なし |
| **JOL自己評価** | 学習者自身のメタ認知支援 | LocalStorageのみ | ❌ なし |
| **Quiz履歴** | 学習者自身の復習支援 | LocalStorageのみ | ❌ なし |

これらの機能は提供するが、**そのデータを運営側が収集・集計することはない**。

### 6.6 除外する指標

以下の指標は明示的に**採用しない**。

| 指標 | 除外理由 |
| :--- | :--- |
| **JOL較正精度** | 学習者のメタ認知状態の追跡が必要 → プライバシー侵害 |
| **間違いからの学び率** | 学習者の失敗と内省の追跡が必要 → プライバシー侵害 |
| **自己説明率（運営KPI）** | 学習者の認知行動の追跡が必要 → プライバシー侵害 |
| **LP完了率（運営KPI）** | 学習者の進捗追跡が必要 → プライバシー侵害 |
| **Quiz正答率** | 学習者の知的能力の推定に繋がる → プライバシー侵害 |
| **滞在時間** | 学習困難の推定に繋がる可能性 → プライバシー侵害 |

> **注**: これらの機能を提供しないのではなく、**運営側がデータを収集しない**ということ。機能自体は学習者のために提供され、データは学習者のデバイスにのみ保存される。

## 7. 公正性指標

CORE_FOUNDATIONS.mdで定義された教育的公正（Educational Equity）を測定する。

### 7.1 設計原則

| 原則 | 説明 |
| :--- | :--- |
| **アクセスの多様性** | 異なる身体能力、言語、技術環境からのアクセスを測定 |
| **理論的根拠** | [CORE_FOUNDATIONS.md §1](./CORE_FOUNDATIONS.md)（教育的公正）に基づく |

> **注**: 「結果の公正性」（学習成果の格差監視）は重要な原則だが、プライバシーを侵害せずに測定する方法がないため、現時点では指標化しない。将来、プライバシー保護と両立する代理指標が見つかれば再検討。

### 7.2 アクセシビリティスコア

| 項目 | 内容 |
| :--- | :--- |
| **定義** | Lighthouseアクセシビリティスコアの全ページ平均 |
| **測定方法** | CI/CDパイプラインでのLighthouse CI |
| **頻度** | デプロイ毎（継続的） |
| **目標** | 100（WCAG 2.2 AA準拠） |

### 7.3 アクセシビリティ改善の間接指標

| 項目 | 内容 |
| :--- | :--- |
| **定義** | アクセシビリティ関連のフィードバック・誤り報告数 |
| **測定方法** | GitHub Issues（`accessibility`ラベル）、お問い合わせフォーム |
| **頻度** | 月次 |
| **目標** | ゼロ（アクセシビリティ上の障壁報告がない理想状態） |
| **解釈** | 報告があれば優先的に対応。報告の不在は「問題がない」または「報告手段が不十分」のいずれか |

> **設計上の注意**: 従来の「User-Agent分析」は支援技術の種類を推定可能なため採用しない。代わりに、学習者からの自発的なフィードバックに依存する。

### 7.4 多言語コンテンツ到達率

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 翻訳版が存在する記事の割合 |
| **計算式** | 翻訳版を持つ記事数 ÷ 公開記事数 |
| **測定方法** | リポジトリ分析 |
| **頻度** | 月次 |
| **目標（Phase 7 + 1年）** | 日本語のみ（Phase 8で英語展開後に再設定） |

### 7.5 低帯域環境成功率

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 3G相当の低帯域環境でのページ読み込み成功率 |
| **測定方法** | Lighthouseスロットリング設定でのパフォーマンステスト |
| **頻度** | デプロイ毎 |
| **目標** | LCP 4秒以内、成功率99%以上 |

## 8. 持続可能性指標

公共図書館として長期的に運営するための健全性を測定する。

### 8.1 月間運営コスト

| 項目 | 内容 |
| :--- | :--- |
| **定義** | インフラ、ドメイン等の総コスト |
| **構成要素** | Cloudflare（ドメイン、DNS）、開発ツール等 |
| **測定方法** | 会計記録 |
| **頻度** | 月次 |
| **目標（1年目）** | $50/月以下 |
| **目標（将来）** | 寄付・グラントでカバー |

### 8.2 コンテンツ増加率

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 月あたりの新規記事公開数 |
| **測定方法** | GitHub統計、リポジトリ分析 |
| **頻度** | 月次 |
| **目標（1年目）** | 月2記事以上 |

### 8.3 コントリビューター数

| 項目 | 内容 |
| :--- | :--- |
| **定義** | アクティブな貢献者数（執筆者、レビュアー） |
| **測定方法** | GitHub Statistics |
| **頻度** | 四半期 |
| **目標（初期）** | 1名（創設者） |
| **目標（将来）** | 3名以上 |

### 8.4 寄付・グラント

| 項目 | 内容 |
| :--- | :--- |
| **定義** | 外部からの資金援助 |
| **測定方法** | 会計記録 |
| **頻度** | 年次 |
| **目標（将来）** | 運営コストの50%以上をカバー |

### 8.5 リスク指標

持続可能性を脅かすリスクを監視する。

| 指標 | 定義 | 測定方法 | 目標 |
| :--- | :--- | :--- | :--- |
| **バス係数** | プロジェクトが通常運営できなくなる最小人数 | 創設者依存度の定性評価 | 2名以上（将来） |
| **法的リスク** | 著作権侵害クレーム数 | GitHub Issues、法的通知 | 0件 |

## 9. 除外する指標（明示的に追わない）

以下の指標はIsidoricaの哲学と矛盾するため、意図的に追跡しない。

| 指標 | 除外理由 |
| :--- | :--- |
| **MAU / DAU** | 量より質。公共知は「多くの人が使う」ことが目的ではない |
| **広告収入** | ポリシー違反（広告なし） |
| **ソーシャルシェア数** | バズ狙いは信頼性を損なう |
| **ユーザー獲得コスト** | 有料広告を使わない |
| **ページビュー（単独）** | 単独では意味がない。質的指標と組み合わせる場合のみ参考 |
| **コンバージョン率** | 「コンバージョン」の概念が適用されない |
| **リテンション率（従来定義）** | 「毎日戻ってくる」ことは目標ではない |

## 11. フェーズ別目標値

### 11.1 目標設定アプローチ

多くの指標について、適切な目標値は**ベースライン測定後**に設定する。初期から根拠のない目標を設定すると、歪んだ行動を誘発するリスクがある。

| 目標設定タイプ | 説明 | 適用指標 |
| :--- | :--- | :--- |
| **絶対目標** | 理論的・技術的根拠がある固定値 | アクセシビリティスコア、誤り報告率、読者評価 |
| **ベースライン+改善率** | 測定後に前期比改善率を設定 | 教育機関採用数 |
