## 1. 概要

### 1.1 本文書の目的

本文書は、4次元読者モデルのうち**学習者の状態**に関する2次元を定義する：

| 次元 | 定義 | 性質 |
|:---|:---|:---|
| **K（知識状態）** | 当該ドメインで習得した概念の集合 | ドメイン固有、動的 |
| **M（メタ学習スキル）** | 学習方法自体に関するスキル | 比較的安定 |

### 1.2 K と M の相互作用

K と M は相互に影響し合う：

| 組み合わせ | 学習者の状態 | 必要な支援 |
|:---|:---|:---|
| K₀ + M_low | 前提不足 + 学び方が分からない | **最も手厚い支援**（アコモデーション、学習方略ガイド、外的構造） |
| K₀ + M_high | 前提不足だが自律的に学べる | アコモデーション、最小限の方略ガイド |
| K₁/K₂ + M_low | 内容は理解できるが学習効率が低い | 学習方略ガイド、メタ認知支援 |
| K₁/K₂ + M_high | 自律的に学習できる | 最小限の支援 |

# Part I: K次元（知識状態）

## 2. 形式モデル

### 2.1 知識状態の定義

ユーザー $u$ の知識状態 $K_u$ は、習得した概念の集合として定義される：

$$K_u = \{ c_1, c_2, ..., c_n \}$$

ここで $c_i$ は概念識別子（Isidorica内のトピックID）。

### 2.2 記事の前提

各記事 $A$ は前提知識集合 $P_A$ を持つ：

$$P_A = \{ p_1, p_2, ..., p_m \}$$

これはFrontmatterの `requires` フィールドで宣言される：

```yaml
---
title: ベイズ推定入門
requires:
  - conditional-probability
  - bayes-theorem
---
```

### 2.3 適正性条件

記事 $A$ の読者に対する**適正性条件**：

$$P_A \subseteq K_u$$

すなわち、記事の前提が全てユーザーの知識状態に含まれていること。

### 2.4 知識状態の更新

ユーザーが記事 $A$ を「習得」した場合、知識状態は更新される：

$$K_u' = K_u \cup \{ A \}$$

### 2.5 理論的背景：動的意味論

| 概念 | 言語学的意味 | 学習設計への翻訳 |
|:---|:---|:---|
| **文脈（Context）** | 談話参加者が共有する情報の集合 | ユーザーが習得した概念の集合 |
| **文脈更新（Context Update）** | 新情報により文脈が変化 | 新しい概念の習得による知識状態の更新 |
| **前提（Presupposition）** | 発話理解に事前に必要な情報 | 記事理解に必要な前提知識 |
| **適正性条件（Felicity Condition）** | 前提が満たされなければ発話は不適切 | 前提知識なしでは学習が困難 |
| **アコモデーション（Accommodation）** | 前提未充足でも暗黙的に受け入れる | 簡略説明による補完 |

## 3. K次元の機能設計

### 3.1 前提充足チェック

#### チェックフロー

```
記事アクセス時
  ↓
前提 P_A を取得
  ↓
ユーザー知識状態 K_u と照合
  ↓
[全て充足] → 通常表示
[一部未充足] → アコモデーション支援を表示
[大部分未充足] → 警告 + 推奨学習パスを提示
```

#### 未充足時の対応（アコモデーション）

| 未充足レベル | 対応 |
|:---|:---|
| **軽微（1-2個）** | Toggletipで簡略説明を提供 |
| **中程度（3-4個）** | 「この記事の前に読むことをお勧めします」バナー |
| **重大（5個以上）** | 「この記事は難しいかもしれません」警告 + 学習パス提案 |

### 3.2 習得判定

#### 習得の基準

| 方法 | 説明 | 信頼度 |
|:---|:---|:---|
| **確認クイズ通過** | 記事末尾のクイズで閾値以上のスコア | 高 |
| **自己申告** | 「この記事を理解しました」ボタン | 中（過信の可能性） |
| **時間閾値** | 記事に一定時間滞在 | 低（参考情報のみ） |

#### JOL較正フィードバック

```
記事読了時
  ↓
「この記事をどの程度理解したと思いますか？」（1-5スケール）
  ↓
[後日] 確認クイズ実施
  ↓
「あなたの自己評価: 4/5、実際のスコア: 60%」
```

### 3.3 学習パス推薦

「次に学ぶべき記事」は、以下の条件を満たすものから選定：

1. **前提が充足**: $P_A \subseteq K_u$
2. **未習得**: $A \notin K_u$
3. **関連性が高い**: 現在読んでいる記事との Semantic Relation が強い

#### DAG（有向非巡回グラフ）としての学習パス

概念間の `requires` 関係はDAGを形成する：

```
確率の基礎 → 条件付き確率 → ベイズの定理 → ベイズ推定
                            ↘
                              マルコフ連鎖
```

ユーザーの知識状態 $K_u$ に基づき、「到達可能な次のノード」を推薦。

# Part II: M次元（メタ学習スキル）

## 4. M次元の設計原則

### 4.1 問題の構造

| 理論 | 問題の記述 | 設計への含意 |
|:---|:---|:---|
| **合理的教育学（Bourdieu）** | 教育的不平等の多くは「暗黙の前提」によって生じる | 暗黙知を明示化する必要がある |
| **言語コード論（Bernstein）** | 「精密コード」を家庭で獲得していない層は不利 | 学術的談話の規則を明示する必要がある |
| **意欲格差（苅谷）** | 「なぜ学ぶのか」が自明でない層が存在する | 学習の意味を明示する必要がある |

### 4.2 設計原則

| 原則 | 説明 |
|:---|:---|
| **統合原則** | メタ学習支援はコンテンツに統合（別モジュールにしない） |
| **可視性原則** | 暗黙の評価基準・前提・期待を明示的に言語化 |
| **自己認識支援** | 学習者が自分の状態を正確に認識できるよう支援 |

---

## 5. M次元の機能設計

### 5.1 コンテンツ統合型メタ学習支援

各記事に以下の要素を統合する：

#### Before Reading（読む前に）

| 要素 | 内容 | 理論的根拠 |
|:---|:---|:---|
| **目的の明示** | 「この記事を読むと何ができるようになるか」 | 意欲格差への対処 |
| **前提の明示** | 「この記事を読むために必要な知識」 | ZPD |
| **読み方のヒント** | 「この記事の効果的な読み方」 | 暗黙知の明示化 |

```markdown
:::before-reading
## この記事を読む前に

**目的**: この記事を読むと、〇〇と△△の違いを説明できるようになります。

**前提知識**: [[基本的な確率の概念]] を理解していることを前提としています。

**読み方のヒント**: まずは「なぜこの式が必要なのか」という直観を掴むことを
優先してください。
:::
```

#### After Reading（読んだ後に）

| 要素 | 内容 | 理論的根拠 |
|:---|:---|:---|
| **理解度自己評価** | 「どの程度理解したと思いますか？」 | メタ認知較正 |
| **確認質問** | 核心的な理解を確認する質問 | 遅延JOL効果 |
| **次のステップ** | 「次に読むべき記事」 | 学習パスの構造化 |

### 5.2 学習方略ガイド

#### 効果的な方略

| 方略 | 説明 | 有効性の根拠 |
|:---|:---|:---|
| **分散学習** | 間隔を空けて複数回学ぶ | 間隔効果 |
| **検索練習** | 思い出そうとする練習 | テスト効果 |
| **精緻化** | 「なぜか」を自分に問う | 深い処理 |
| **インターリービング** | 類似内容を混ぜて練習 | 転移促進 |

#### 非効果的な方略への警告

| 非効果的方略 | なぜ効果がないか |
|:---|:---|
| **再読** | 流暢性の錯覚を生む |
| **ハイライト** | 深い処理を促さない |
| **徹夜学習** | 記憶の定着に睡眠が必要 |

### 5.3 M次元に応じた支援調整

| M値 | 対応 |
|:---|:---|
| **M_high** | 最小限の支援、メタ学習要素はデフォルト折りたたみ |
| **M_low** | フル提供（Before/After Reading、学習方略ガイド） |

### 5.4 学術的リテラシー支援

学術的文献との付き合い方を明示的に教える：

| 読み方のステップ | 説明 |
|:---|:---|
| 1. 全体を把握 | まず見出しだけを読み、構造を把握 |
| 2. 目的を明確に | 「何を知りたくて読むのか」を意識 |
| 3. 既知と未知を区別 | 知っていることと知らないことを区別 |
| 4. 疑問を持つ | 「なぜ？」「本当に？」と問いながら読む |
| 5. 自分の言葉で言い換え | 理解したことを自分の言葉で表現 |

### 5.5 Directive記法

```markdown
:::before-reading
...
:::

:::after-reading
...
:::

:::learning-strategy
...
:::
```

**表示ロジック**:
- **初回表示**: 全て展開
- **再訪問時**: 折りたたみ状態（ユーザーが明示的に開く）
- **設定**: ユーザーが「メタ学習支援を非表示」を選択可能

# Part III: 共通設計

## 6. データモデル

### 6.1 ユーザー状態スキーマ

```typescript
interface LearnerState {
  // K次元
  acquiredConcepts: {
    conceptId: number;
    acquiredAt: number;      // timestamp
    method: 'quiz' | 'self-report' | 'inferred';
    confidence: number;      // 0-1
  }[];
  jolHistory: {
    conceptId: number;
    selfRating: number;      // 1-5
    actualScore: number | null;
  }[];
  
  // M次元
  metalearningLevel: 'high' | 'low' | null;
  metalearningPrefs: {
    showBeforeReading: boolean;
    showAfterReading: boolean;
    showStrategyGuide: boolean;
  };
}
```

### 6.2 ストレージ戦略

| データ種別 | 保存先 | 理由 |
|:---|:---|:---|
| K（現在状態） | localStorage | 即時アクセス、オフライン対応 |
| K（学習履歴） | IndexedDB | 容量制限なし |
| M設定 | localStorage | 軽量 |

### 6.3 容量分析

| シナリオ | 概念数 | 容量見積もり |
|:---|:---|:---|
| 一般ユーザー | 100〜500 | 5KB〜25KB |
| ヘビーユーザー | 1,000〜2,000 | 50KB〜100KB |

### 6.4 localStorage形式

```typescript
interface LocalStorageData {
  isidorica_k: {
    v: number;           // スキーマバージョン
    l: number[];         // learned概念ID（K₂）
    d: number[];         // developing概念ID（K₁）
    updated: number;
  };
  isidorica_m: 'high' | 'low' | null;
  isidorica_m_prefs: {
    before: boolean;
    after: boolean;
    strategy: boolean;
  };
}
```

### 6.5 容量最適化手法

**A. 概念IDの数値化**（削減率60-70%）

```javascript
// 非効率（文字列ID）
{ "ナッシュ均衡": { status: "K2", at: 1704567890 } }  // ~50 bytes

// 効率的（数値ID）
{ "l": [123, 456], "d": [789] }  // ~20 bytes
```

**B. 圧縮（オプション）**: LZ-string等のクライアントサイド圧縮を適用可能。通常規模では不要。

### 6.6 IndexedDB（履歴保存）

```typescript
interface LearningHistoryEntry {
  conceptId: number;
  action: 'started' | 'completed' | 'revisited';
  timestamp: number;
  duration: number | null;  // 滞在時間（秒）
  jolRating: number | null; // 自己評価（1-5）
}
```

### 6.7 アカウント同期（オプション）

| 状態 | 動作 |
|:---|:---|
| **匿名ユーザー** | localStorage + IndexedDBのみ。デバイス限定。 |
| **登録ユーザー** | 上記 + サーバー（Cloudflare KV）に定期同期 |

**同期タイミング**:
- ログイン時: サーバー → ローカルにマージ
- 学習完了時: ローカル → サーバーに差分送信
