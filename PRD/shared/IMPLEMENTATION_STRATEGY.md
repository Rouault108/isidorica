理論から直接導出される原則（広告排除、アクセス無制約、知識開放、AI証言制約）は [DESIGN_PRINCIPLES.md カテゴリG](./DESIGN_PRINCIPLES.md#8-カテゴリ-g-事業持続可能性) を参照。

## 1. 資金・持続可能性戦略

> **前提**: 広告なし（G-1）、ペイウォールなし（G-2）、オープンライセンス（G-3）は設計原則として確定済み。本セクションでは、これらの制約下での戦略的選択を記述する。

### 1.1 教育の市場化と持続可能性の課題

#### 教育の市場化への批判的認識

フィリップ・ブラウンは、現代の教育システムが**「ペアレントクラシー（Parentocracy）」**―子どもの教育機会が親の富と意向（wealth and wishes）によって決まる社会―に変質していると批判した [Brown 1990]。教育の市場化は、「良い教育」を**購入可能な商品**に変え、**地位財（Positional Good）**としての競争を激化させる。

苅谷が指摘した「ゆとり教育のパラドックス」において、**塾**が決定的な役割を果たしていた。学校が緩くなっても、経済資本を持つ家庭は有料の塾で学習時間を維持した。

一方で、教育の市場化には批判すべき側面がある一方、競争が質の向上やイノベーションを促すという正当な議論も存在する [Friedman 1962; Chubb & Moe 1990]。Isidoricaは市場化を全面否定するのではなく、**公共財としての知識提供**と**持続可能な運営**の両立を模索する。

#### 「無料」の真のコスト

教育プラットフォームの「無料で質の高いコンテンツ」というミッションには、構造的な緊張がある。

| 視点 | 実態 |
|:---|:---|
| **ユーザーにとっての「無料」** | アクセスに経済的障壁がない |
| **制作者にとっての「コスト」** | コンテンツの作成、レビュー、維持には専門知識と時間が必要 |

> **正直な認識**: 「無料」は「コストゼロ」を意味しない。現在のモデルは、創設者やレビュワーの貢献に依存しており、長期的な持続可能性には課題がある。

#### 長期的な持続可能性への道筋

| アプローチ | 説明 | 課題 |
|-----------|------|------|
| **機関連携** | 教育機関との連携により、制度的支援を獲得する | 機関の意思決定に依存 |
| **寄付・支援** | ミッションに共感する個人・団体からの支援 | 資金の不安定性 |
| **貢献者コミュニティ** | 貢献がポートフォリオやキャリアに価値を持つエコシステムの構築 | 全員には適用不可 |
| **公的資金への提言** | 教育を公共インフラとして支援する政策への働きかけ | 政治的意志に依存 |

> **共通の課題**: この持続可能性の課題は、Isidoricaに固有の問題ではなく、**教育を公共財として提供しようとするすべてのプロジェクトに共通する根本的困難**である。Wikipedia、Khan Academy、MIT OpenCourseWareなど、先行事例もこの課題と格闘し続けている。

### 1.2 収入源の優先順位

**戦略的判断**:

| 収入源 | 優先度 | 備考 |
|:---|:---|:---|
| **個人寄付（定額/一時）** | 高 | Wikipedia/SEPモデル。コミュニティのオーナーシップ形成 |
| **機関グラント** | 高 | 研究助成、教育財団等 |
| **コスメティック課金** | 中 | アカウントカスタマイズ。後述の制約付き |
| **機関スポンサー** | 中 | 大学・図書館等との提携。広告ではない形態 |
| **API/機関ライセンス** | 低 | 機関向け有料サービス。本体コンテンツは無料維持 |

**棄却した選択肢**:

| 選択肢 | 棄却理由 |
|:---|:---|
| **広告モデル** | G-1原則（注意経済排除）により棄却 |
| **サブスクリプションモデル** | G-2原則（アクセス無制約）により棄却 |
| **VC資金調達** | 成長圧力が公共財としての性質と矛盾 |
| **クラウドファンディング** | 初期は検討可能だが、持続的収入源としては不十分 |

### 1.3 コスメティック課金（Discordモデル）の検討

**理論的分析**:

| 観点 | 評価 | 根拠 |
|:---|:---|:---|
| **情報正義** | ✅ 矛盾しない | 知識アクセスに影響しない（装飾は私的財、知識は公共財） |
| **教育的公正** | ⚠️ 注意が必要 | 学習成果への影響なし。ただし可視的階層化のリスク |
| **動機づけ** | ⚠️ 設計次第 | 装飾獲得が目的化しないよう制御が必要 |

**採用可能な項目**:

| 項目 | 理論的正当性 |
|:---|:---|
| **プロフィールカスタマイズ**（アバター、バナー、カード背景） | 自己表現であり学習に影響しない |
| **貢献者バッジの装飾バリエーション** | 内容（貢献レベル）は同じ、見た目だけ異なる |
| **サポーターマーク** | 「このプラットフォームを支援」の任意表示 |
| **カスタム絵文字作成** | 個人用途限定。Agoraでの過度な使用は制限 |

**避けるべき項目**:

| 項目 | 理論的懸念 |
|:---|:---|
| **学習コンテンツへの影響** | 有料で追加コンテンツ → 情報正義違反 |
| **発言の可視性への影響** | 有料で発言が目立つ → エピステミック不正義 |
| **機能制限の解除** | 有料でファイル上限増加等 → 情報正義的に微妙 |
| **学習進捗バッジの装飾** | 学習成果に金銭が介入 → 教育的公正に反する |
| **UX機能の有料化** | ダークモード等 → 情報正義的に微妙 |

**設計原則**:

| 原則 | 説明 |
|:---|:---|
| **コンテンツと装飾の分離** | 知識（公共財）と装飾（私的財）を明確に分離 |
| **Pay-to-Winの排除** | 課金が学習成果や発言権に影響しないことを保証 |
| **控えめな表示** | 有料装飾が無料ユーザーを圧迫しない設計 |
| **透明性** | 収益の使途を公開 |

### 1.4 運営コスト最小化

**戦略的判断**:

| 判断 | 内容 |
|:---|:---|
| **静的サイト優先** | Astro SSGによる低コストホスティング |
| **コミュニティ維持** | ボランティア編集者による持続可能な運営 |
| **インフラ選択** | Cloudflare Pages/R2による低コスト・高性能配信 |

## 2. AI戦略

> **前提**: AI司書を持たない（G-4）は設計原則として確定済み。本セクションでは、AI活用の戦略的選択を記述する。

### 2.1 AIへのスタンス

**戦略的判断**:

| 判断 | 内容 | 理由 |
|:---|:---|:---|
| **AIによる形式査読** | 編集支援ツールとしてAIを活用 | 人間の判断を支援するが、置き換えない |
| **AI-Readable設計** | コンテンツをAIが参照しやすい形式で公開 | 他のAIシステムがIsidoricaを情報源として引用できるようにする |
| **AI翻訳の活用** | 下書きとして活用可、最終は人間がレビュー | 機械翻訳のみでの公開は禁止 |
| **AI画像生成** | 図解作成の補助として容認、明示必須 | オリジナル作成または出典明示 |

**棄却した選択肢**:

| 選択肢 | 棄却理由 |
|:---|:---|
| **AI司書（チャットボット）** | G-4原則（証言の認識論）により棄却 |
| **AI生成コンテンツ全面禁止** | 編集効率を著しく低下させる |
| **AI生成コンテンツ全面許可** | 品質管理・責任の所在が曖昧になる |
| **AI単独での翻訳公開** | 証言の認識論的責任が不明確 |

### 2.2 AI生成コンテンツポリシー

**戦略的判断**:

| 項目 | 方針 |
|:---|:---|
| **AI生成コンテンツの掲載** | 禁止しないが、人間による検証・編集を必須とする |
| **AI利用の開示** | 義務ではないが、任意開示を歓迎する（後述） |
| **出典の優先** | AI生成ではなく、人間の著者による一次文献を優先 |
| **署名責任原則** | レビュワー（署名者）がコンテンツの全責任を負う |

**AI利用開示に関する詳細**:

| 観点 | 判断 | 理由 |
|:---|:---|:---|
| **義務化しない理由** | 執行不可能（AIかどうかをレビュワーは検知できない） | 誠実な執筆者のみが不利になる構造を避ける |
| **歓迎する理由** | 透明性への貢献として評価 | 広義の透明性原則に資する |
| **責任の所在** | 生成手段に関わらずレビュワーが全責任を負う | 証言の認識論と整合 |

## 3. 言語戦略

### 3.1 対象言語

**戦略的判断**:

| フェーズ | 対象言語 | 理由 |
|:---|:---|:---|
| **Phase 1** | 日本語のみ | 創設チームの言語。質の確保を優先 |
| **Phase 2以降** | 英語への展開検討 | グローバルなアクセシビリティ。翻訳ではなく並行コンテンツ |

### 3.2 翻訳方針

**理論的背景**（[TRANSLATION_FOUNDATIONS.md](./TRANSLATION_FOUNDATIONS.md)参照）:
- スコポス理論：翻訳の目的が方法を規定する
- 等価論の限界：完全等価は達成不可能

**戦略的判断**:

| 判断 | 内容 |
|:---|:---|
| **翻訳より並行** | 英語版は翻訳ではなく、英語ネイティブによる並行執筆を優先 |
| **概念の対応関係** | 同一概念IDで日英版をリンク。完全翻訳ではなく概念レベルで対応 |
| **文化的適応** | 例示・文脈は言語圏に適応。直訳を避ける |

**棄却した選択肢**:

| 選択肢 | 棄却理由 |
|:---|:---|
| **最初から英語優先** | 品質管理が困難。創設チームの強みを活かせない |
| **機械翻訳のみでの展開** | 品質保証ができない |
| **全言語同時展開** | リソース不足。品質分散のリスク |

## 4. コミュニティ戦略

### 4.1 貢献者採用

**理論的背景**（[AGORA_FOUNDATIONS.md §3.1](../library/AGORA_FOUNDATIONS.md)参照）:
- 正統的周辺参加（Lave & Wenger）：周辺から中心への段階的移行
- 参加障壁の理論：高障壁は品質を担保するが参加を阻害

**戦略的判断**:

| 判断 | 内容 |
|:---|:---|
| **閲覧は完全オープン** | アカウント不要で全コンテンツ閲覧可能 |
| **投稿は最小障壁** | アカウント作成のみ。実名・履歴書は不要 |
| **権限は段階的** | 貢献実績に応じて編集・レビュー権限を付与 |
| **専門家の招聘** | 重要領域では専門家に執筆を依頼。コミュニティ編集と並行 |

**棄却した選択肢**:

| 選択肢 | 棄却理由 |
|:---|:---|
| **完全オープン（Wikipedia型）** | 品質管理が困難。初期は統制が必要 |
| **完全クローズド（Scholarpedia型）** | 参加障壁が高すぎる。持続可能性に懸念 |
| **実名必須** | 専門家以外の貢献を阻害。プライバシー懸念 |

### 4.2 ガバナンス

**戦略的判断**:

| 判断 | 内容 |
|:---|:---|
| **透明な意思決定** | 主要な決定はAgora等で公開議論 |
| **コミュニティ代表** | 一定の貢献者から選出される運営委員会 |
| **創設者の役割** | 初期はガイドラインを設定。漸進的に権限をコミュニティに移譲 |

## 5. 技術戦略

### 5.1 アーキテクチャ選択

**戦略的判断**:

| 判断 | 選択 | 理由 |
|:---|:---|:---|
| **フレームワーク** | Astro (SSG) | 静的サイトによる低コスト・高速配信 |
| **UIライブラリ** | Lit (Web Components) | フレームワーク非依存。長期維持可能性 |
| **ホスティング** | Cloudflare Pages | エッジ配信。低コスト。高可用性 |
| **アセット** | Cloudflare R2 | 画像・動画の低コスト配信 |
| **Laboratory** | Client-side WASM | サーバー負荷なし。オフライン対応可能 |

**棄却した選択肢**:

| 選択肢 | 棄却理由 |
|:---|:---|
| **Next.js/Remix（SSR）** | サーバーコストが高い。静的コンテンツには過剰 |
| **React/Vue** | フレームワークロックイン。100年維持には不向き |
| **サーバーサイドLaboratory** | コストが高い。オフライン対応不可 |
| **AWS/GCP** | Cloudflareより高コスト。複雑性が高い |

### 5.2 オープンソース方針

**戦略的判断**:

| 判断 | 内容 |
|:---|:---|
| **コードはオープン** | プラットフォームコードはMITライセンスで公開 |
| **コンテンツはCC** | 記事コンテンツはCC BY-SAで公開 |
| **フォーク歓迎** | 他者によるフォーク・派生プロジェクトを歓迎 |

> **参照**: 技術選定の詳細は [ARCHITECTURE_DECISION_RECORD.md](../engine/technical/ARCHITECTURE_DECISION_RECORD.md) を参照。

### 5.3 学習分析とプライバシー方針

**理論的背景**（[LEARNING_FOUNDATIONS.md §7.5](./LEARNING_FOUNDATIONS.md)参照）:
- 学習分析（Learning Analytics）は学習改善に寄与しうるが、プライバシーリスクを伴う
- DELICAATEチェックリスト（Drachsler & Greller, 2016）は学習分析の倫理的実装のフレームワークを提供

#### 5.3.1 基本方針

**戦略的判断**:

| 判断 | 内容 | 理由 |
|:---|:---|:---|
| **デフォルト非収集** | サーバーサイドでの学習データ収集をデフォルトで行わない | プライバシー保護を優先 |
| **クライアントサイド優先** | パーソナライゼーション機能はローカルストレージで実装 | データがデバイスを離れない |
| **オプトイン原則** | 将来的な拡張時も、サーバーサイド収集は明示的同意を前提 | 自律性の尊重 |

#### 5.3.2 段階的アプローチ

| 段階 | 実装 | データ場所 | プライバシーリスク |
|:---|:---|:---|:---|
| **Tier 0（現行）** | データ収集なし | — | 最低 |
| **Tier 1** | クライアントサイドのみ（進捗追跡、間隔反復） | ユーザーデバイス | 低 |
| **Tier 2（検討中）** | オプトイン分析（同意取得） | サーバー | 中（要管理） |

#### 5.3.3 オプトイン学習分析の条件（Tier 2実装時）

将来、ユーザーからの要望に応じてオプトイン学習分析を実装する場合、以下のDELICATEチェックリストを遵守する。

| 項目 | 説明 | 実装要件 |
|:---|:---|:---|
| **D - Determination** | 決定権 | ユーザー自身がデータ収集ON/OFFを決定 |
| **E - Explain** | 説明 | 収集データ・利用目的・保持期間を明確に説明 |
| **L - Legitimate** | 法的根拠 | GDPR、個人情報保護法に準拠 |
| **I - Involve** | 関与 | データ活用方針にユーザーが意見できる仕組み |
| **C - Consent** | 同意 | 明示的オプトイン（チェックボックス等） |
| **A - Anonymize** | 匿名化 | 可能な限り匿名化/仮名化を実施 |
| **T - Technical** | 技術的保護 | 暗号化、アクセス制御、定期監査 |
| **E - External** | 外部アクセス | 第三者への非開示。研究目的でも匿名化必須 |

#### 5.3.4 ユーザーの自律性と選択

**論点**:
- 一部の学習者は、データ活用によるパーソナライゼーションを**望む**可能性がある
- 「データを使わせない」という方針も一種のパターナリズムになりうる
- 自己決定理論（Deci & Ryan）に基づけば、学習者に選択権を与えることが自律性を支持する

**戦略的判断**:

| 判断 | 内容 |
|:---|:---|
| **選択権の提供** | ユーザーが自身のデータ活用レベルを選択できる設計 |
| **デフォルトはプライバシー優先** | オプトインが未選択の場合、データ収集しない |
| **透明性** | 各選択肢のリスクとベネフィットを明示 |

#### 5.3.5 棄却した選択肢

| 選択肢 | 棄却理由 |
|:---|:---|
| **オプトアウト方式** | デフォルトで収集はプライバシー侵害リスクが高い |
| **全ユーザー強制収集** | 自律性の侵害。プロジェクト理念に反する |
| **学習分析の完全排除** | パーソナライゼーションを望むユーザーへの非対応 |
| **第三者分析サービス利用** | データ管理が困難。プライバシーリスク増大 |

#### 5.3.6 学習分析の有料化に関する方針

**論点**:
学習分析機能を有料化することは、運営コストの回収に寄与する一方、教育的公正の観点から懸念がある。

**理論的分析**:

| 観点 | 分析 | 判定 |
|:---|:---|:---|
| **情報正義（Floridi）** | 学習分析は「知識」ではなく「最適化ツール」。ただし学習効率に差が生じる | グレーゾーン |
| **教育的公正（Sen）** | 有料化は既存の教育格差を強化するリスク。学習困難層への影響 | 懸念あり |
| **コスト正当性** | クライアントサイド分析は追加コストなし、サーバーサイドは実コスト発生 | 区分が必要 |

**戦略的判断**:

| 区分 | 判断 | 理由 |
|:---|:---|:---|
| **Tier 1（クライアントサイド）** | **無料であるべき** | 追加コストなし。有料化の正当性が欠如 |
| **Tier 2（サーバーサイド）** | **条件付きで検討可能** | 実コストが発生する場合、以下の条件を満たす場合に限る |

**Tier 2有料化の条件（仮に検討する場合）**:

| 条件 | 説明 |
|:---|:---|
| **経済困難者への無料提供** | 申請ベースで無料アクセスを保証 |
| **教育機関への無料提供** | 学校・大学・図書館等への無料ライセンス |
| **コスト回収のみ** | 利益を上げない（実費ベースの価格設定） |
| **段階的無料化** | 寄付収入で運営費が賄えれば無料化へ移行 |
| **透明なコスト開示** | 価格の根拠となるコスト構造を公開 |

**代替案**:

| 案 | 説明 |
|:---|:---|
| **寄付ベース運営** | 学習分析は全員無料、運営は寄付で賄う |
| **機関スポンサー** | 大学・財団がスポンサーとなり一般ユーザーは無料 |
| **クライアントサイドのみ実装** | サーバーサイド分析を実装しない |

> **結論**: 学習分析の有料化は教育的公正の理念と緊張関係にある。クライアントサイド分析は無料で提供し、サーバーサイド分析は実コストが発生する場合に限り、経済困難者への配慮と透明なコスト開示の下でのみ検討可能とする。

> **参照**: 倫理的フレームワークは [CORE_FOUNDATIONS.md](./CORE_FOUNDATIONS.md) を参照。
> **理論的基盤**: [LEARNING_FOUNDATIONS.md §7.5](./LEARNING_FOUNDATIONS.md)（学習分析）

| **相互参照** | 他プラットフォームへのリンク・引用を積極的に行う | 囲い込みではなく開放性 |
| **AI情報源** | 他のAIシステムがIsidoricaを引用できる設計 | 知識の流通を促進 |
