## 1. 記事構造の設計

### 1.1 設計原則

記事構造は以下の設計原則から演繹される。原則の詳細は [DESIGN_PRINCIPLES.md](./DESIGN_PRINCIPLES.md) を参照。

#### カテゴリ B: 認知基盤

| 原則 | 記事構造への適用 |
| :--- | :--- |
| **B-1（先行知識活性化）** | Learning Layer Step 1: 読者の予測を活性化し、予測誤差を生成する |
| **B-2（身体接地）** | Learning Layer Step 3: 操作・体験を通じた理解を促す要素の埋め込み |
| **B-3（メタ認知較正）** | Mastery Layer Step 6: JOLと実際のパフォーマンスのフィードバック |
| **B-4（遠隔転移促進）** | Mastery Layer Step 4: 抽象構造の明示、複数文脈での提示 |
| **B-5（認知的徒弟制）** | 3層構造全体: モデリング→足場かけ→フェーディングの構造 |
| **B-6（望ましい困難）** | Mastery Layer Step 6: 検索練習、交互練習による記憶強化 |
| **B-7（注意資源管理）** | 記事長制限、セクション間の休憩ポイント（Soft Fascination） |
| **B-8（処理整合性）** | Learning Layer Step 3, Mastery Layer Step 6: 学習時と使用時の処理プロセスの一致（TAP） |
| **B-9（マルチメディア学習）** | 全層: テキスト・図の空間統合、装飾排除、段階的提示、自己ペース制御 |

#### カテゴリ C: 社会・公正

| 原則 | 記事構造への適用 |
| :--- | :--- |
| **C-1（暗黙知明示化）** | Before Reading: 前提知識、学習方略の明示 |
| **C-2（外的構造）** | 学習パス、進捗の可視化、セクション構造 |
| **C-3（学習意味明示）** | Mastery Layer Step 5: 学習の意味と価値の明示、価値の内在化促進 |

#### カテゴリ D: 動機・感情

| 原則 | 記事構造への適用 |
| :--- | :--- |
| **D-1（生産的混乱）** | Learning Layer Step 1-2: 反直観的事例による認知的不均衡の誘発 |
| **D-3（認識的感情）** | Learning Layer Step 1-2: 驚き→好奇心→Aha!体験の設計 |
| **D-4（フロー促進）** | 層間移行: スキル-挑戦バランスの動的調整 |
| **D-5（興味発達）** | Learning→Mastery: 状況興味から個人興味への発達経路 |

#### カテゴリ E: 知識・認識

| 原則 | 記事構造への適用 |
| :--- | :--- |
| **E-2（認識的多様性）** | Mastery Layer Step 4: 複数視点・解釈の意図的提示 |
| **E-3（認識的信念涵養）** | 全層: 知識の暫定性・改訂可能性の明示 |

#### カテゴリ F: 辞書固有

| 原則 | 記事構造への適用 |
| :--- | :--- |
| **F-1（ユーザーニーズ適応）** | P×Tによる層の展開ロジック: アクセス目的と時間制約に応じた情報提供 |
| **F-2（意味フレーム明示）** | Core Layer: 概念の適用範囲、前提条件の明示 |

### 1.2 構造の必要性

#### なぜ構造が必要か

以下の設計原則が**順序**を要求するため、何らかの構造は必要である：

| 原則 | 順序を要求する理由 |
| :--- | :--- |
| **B-1（先行知識活性化）** | 説明の**前**に予測を活性化する必要がある |
| **B-5（認知的徒弟制）** | モデリング→足場かけ→フェーディングの順序 |
| **D-1（生産的混乱）** | 混乱は**解決の前**に生じる必要がある |
| **D-3（認識的感情）** | 驚き→好奇心→Aha!の感情的経路 |
| **C-1（暗黙知明示化）** | 本文**前**に前提知識を明示 |

#### なぜ3層構造か

設計原則から導出される要件を整理すると、3つの異なる**提供条件**を持つ層が必要であることが分かる：

| 層 | 提供条件 | 対応する設計原則 |
| :--- | :--- | :--- |
| **Core Layer** | 全アクセスで常に提供 | F-1（全ユーザーニーズへの最低限の対応）、F-2（意味フレーム） |
| **Learning Layer** | P_cognitive または T > T_quick | B-1, B-2, B-8, D-1, D-3（学習プロセスの促進） |
| **Mastery Layer** | T_extended | B-3, B-4, B-6, C-3, D-5, E-2（深い理解と転移） |

この3層構造は以下の設計原則を同時に満たす：

- **F-1（ユーザーニーズ適応）**: アクセス目的（P次元）に応じて必要な層のみを提供
- **B-9（マルチメディア学習・分割原則）**: 学習者のペースで段階的に情報を展開
- **B-7（注意資源管理）**: 不要な情報を排除し認知負荷を最適化

### 1.3 3層構造（Layered Structure）

```
┌─────────────────────────────────────────────────────────┐
│ Core Layer（全アクセスで提供）                           │
│   - TL;DR、定義（F-1, F-2）                              │
└─────────────────────────────────────────────────────────┘
                          ↓ P_cognitive / T > T_quick
┌─────────────────────────────────────────────────────────┐
│ Learning Layer（学習目的時に展開）                       │
│   Step 1: 先行知識活性化 ← B-1, D-1, D-3                    │
│   Step 2: モデル更新 ← 説明本文                         │
│   Step 3: 体験的検証 ← B-2, B-8（Laboratory）           │
└─────────────────────────────────────────────────────────┘
                          ↓ T_extended
┌─────────────────────────────────────────────────────────┐
│ Mastery Layer（深い学習時に展開）                        │
│   Step 4: 抽象化・転移 ← B-4, E-2                       │
│   Step 5: 意味統合 ← C-3, D-5                           │
│   Step 6: メタ認知統合 ← B-3, B-6                       │
└─────────────────────────────────────────────────────────┘
```

### 1.4 各層の詳細

#### Core Layer（基盤層）

| 項目 | 内容 |
| :--- | :--- |
| **提供条件** | 全アクセスで常に提供 |
| **目的** | 最小限の情報で読者のニーズを満たす |
| **構成要素** | TL;DR、形式的定義、意味フレーム |

**対応原則**:
- **F-1（ユーザーニーズ適応）**: P_receptionでも必要な最小情報を提供
- **F-2（意味フレーム明示）**: 概念の適用範囲、前提条件を明示

**設計パターン**:
```
:::core-layer
## TL;DR

[概念の要約（2-3文）]

## 定義

[形式的定義]

| 適用範囲 | [どのような場面で使うか] |
| 前提知識 | [[前提概念A]], [[前提概念B]] |
| 関連概念 | [[関連概念A]], [[関連概念B]] |
:::
```

#### Learning Layer（学習層）

| 項目 | 内容 |
| :--- | :--- |
| **提供条件** | P = P_cognitive、または T > T_quick |
| **目的** | 概念の深い理解を促進 |
| **構成要素** | 先行知識活性化→モデル更新→体験的検証 |

##### Step 1: 先行知識活性化（Prediction Activation）

| 項目 | 内容 |
| :--- | :--- |
| **目的** | 読者の既存メンタルモデルを活性化し、予測誤差の準備をする |
| **理論的根拠** | 先行知識活性化原則（B-1）、有意味学習（Ausubel） |
| **構成要素** | 導入シナリオ、予測を促す問い、直観的予想の言語化 |

**設計パターン**:
```
:::learning-step-1
## あなたならどう考える？

[具体的なシナリオの提示]

この状況で、何が起こると思いますか？
- [ ] 選択肢A
- [ ] 選択肢B
- [ ] 選択肢C

> 💡 予測を立てることで、学習効果が高まります
:::
```

**対応原則**: B-1（先行知識活性化）、D-1（生産的混乱）、D-3（認識的感情）

##### Step 2: モデル更新（Model Update）

| 項目 | 内容 |
| :--- | :--- |
| **目的** | 概念の正式な説明と、既存モデルの修正を促す |
| **理論的根拠** | 予測誤差による学習、概念変化理論 |
| **構成要素** | 予測との差分の明示、なぜ直観が外れるかの説明、詳細説明 |

**設計パターン**:
```
:::learning-step-2
## 実際には...

[予測誤差の提示]

### なぜ直観が外れるのか

[認知バイアスやメンタルモデルの限界の説明]

### 詳細説明

[概念の詳細な説明]
:::
```

**対応原則**: D-1（生産的混乱）、E-3（認識的信念涵養）

##### Step 3: 体験的検証（Experiential Verification）

| 項目 | 内容 |
| :--- | :--- |
| **目的** | 操作・介入を通じて概念を体験的に理解する |
| **理論的根拠** | 身体接地（B-2）、4E認知、能動的推論 |
| **構成要素** | Laboratory埋め込み、パラメータ操作、結果観察 |
| **適用条件** | 操作可能な概念のみ（抽象概念には非適用） |

**設計パターン**:
```
:::learning-step-3
## 実際に試してみよう

:::laboratory{id="concept-exploration"}
[Laboratoryコンポーネントの埋め込み]
:::

> 🔬 パラメータを変えると、結果がどう変わるか観察してください
:::
```

**対応原則**: B-2（身体接地）、B-8（処理整合性）

> **詳細設計**: [LABORATORY_DESIGN.md](../engine/LABORATORY_DESIGN.md) を参照

---

#### Mastery Layer（習熟層）

| 項目 | 内容 |
| :--- | :--- |
| **提供条件** | T = T_extended |
| **目的** | 転移可能な深い理解と長期記憶への定着 |
| **構成要素** | 抽象化・転移→意味統合→メタ認知統合 |

##### Step 4: 抽象化・転移（Abstraction & Transfer）

| 項目 | 内容 |
| :--- | :--- |
| **目的** | 具体例から抽象構造を抽出し、他領域への転移を促す |
| **理論的根拠** | 遠隔転移理論（B-4）、類推的符号化（Gentner） |
| **構成要素** | 深層構造の明示、複数文脈での適用例、変奏曲的演習 |

**設計パターン**:
```
:::mastery-step-4
## この概念の本質

### 抽象構造

[表面的特徴を超えた深層構造の説明]

### 他分野での適用

| 分野 | 適用例 |
| :--- | :--- |
| 経済学 | ... |
| 生物学 | ... |
| 日常生活 | ... |

> 🔗 同じ構造が異なる文脈で現れることを理解することで、転移が促進されます
:::
```

**対応原則**: B-4（遠隔転移促進）、B-6（望ましい困難）、E-2（認識的多様性）

##### Step 5: 意味統合（Meaning Integration）

| 項目 | 内容 |
| :--- | :--- |
| **目的** | 概念の社会的・歴史的文脈と、学習者にとっての意義を明示 |
| **理論的根拠** | 学習意味明示（C-3）、興味の発達モデル（Hidi & Renninger） |
| **構成要素** | 発見の歴史、社会的インパクト、学習者の生活との関連 |

**設計パターン**:
```
:::mastery-step-5
## なぜこれを学ぶのか

### 歴史的背景

[概念が発見・発展した経緯]

### 社会的影響

[この概念が社会にどのような影響を与えているか]

### あなたの生活との関連

[学習者の日常生活でどのように関連するか]
:::
```

**対応原則**: C-3（学習意味明示）、D-5（興味発達）

##### Step 6: メタ認知統合（Metacognitive Integration）

| 項目 | 内容 |
| :--- | :--- |
| **目的** | 学習内容の振り返りと、メタ認知的較正 |
| **理論的根拠** | メタ認知理論（B-3）、検索練習効果（B-6） |
| **構成要素** | 要約、理解度確認Quiz、JOL較正、次のステップ提示 |

**設計パターン**:
```
:::mastery-step-6
## まとめと確認

### この記事で学んだこと

- [要点1]
- [要点2]
- [要点3]

### 理解度を確認

:::quiz{id="comprehension-check"}
[Quiz問題]
:::

### 次のステップ

- 関連概念: [[関連概念A]], [[関連概念B]]
- 学習パス: [次の記事へ](...)
:::
```

**対応原則**: B-3（メタ認知較正）、B-6（望ましい困難）

### 1.5 P×Tによる層の展開ロジック

| P次元 | T次元 | 展開される層 |
| :--- | :--- | :--- |
| **P_reception** | T_quick | Core のみ |
| **P_reception** | T_extended | Core + Mastery（参照深掘り） |
| **P_cognitive** | T_quick | Core + Learning（Step 1-2のみ） |
| **P_cognitive** | T_extended | **全層** |
| **P_production** | any | Core + 用例・コロケーション（別設計） |
| **P_solving** | any | Core + 手順（別設計） |

> **注**: P_production と P_solving は辞書機能固有であり、[DICTIONARY_CONTENT_DESIGN.md](../library/DICTIONARY_CONTENT_DESIGN.md) で設計する。

## 2. 4次元に基づく適応表示

### 2.1 適応の基本方針

| 次元 | 適応対象 | 適応方法 |
| :--- | :--- | :--- |
| **K（知識状態）** | コンテンツの詳細度、前提説明 | K₀→アコモデーション、K₂→スキップ提案 |
| **M（メタ学習スキル）** | 学習支援の密度 | M_low→Before/After Reading、M_high→最小限 |
| **P（アクセス目的）** | 表示モード | P_cognitive→学習モード、P_reception→参照モード |
| **T（時間制約）** | 要約度 | T_quick→TL;DR優先、T_extended→全Phase表示 |

### 2.2 K次元への適応

> **詳細設計**: [LEARNER_STATE_MODEL.md](./LEARNER_STATE_MODEL.md) §2-3 を参照

| K値 | 適応 | 設計原則 |
| :--- | :--- | :--- |
| **K₀（前提不足）** | アコモデーション提供、前提記事へのリンク | ZPD、足場かけ |
| **K₁（発達中）** | 全Phaseを表示、前提は既知として扱う | 認知負荷最適化 |
| **K₂（習得済）** | スキップ提案、復習オプション | 専門性逆転効果回避 |

**アコモデーションの設計**:
```
:::accommodation
この記事を理解するには「ゲーム理論の基礎」の知識が必要です。

- **簡略説明を見る**: [展開]
- **前提記事から読む**: [ゲーム理論の基礎](...)
- **このまま続ける**: [続行]
:::
```

### 2.3 M次元への適応

> **詳細設計**: [LEARNER_STATE_MODEL.md](./LEARNER_STATE_MODEL.md) §4-5 を参照

| M値 | 適応 | 設計原則 |
| :--- | :--- | :--- |
| **M_high** | 最小限の支援、自律的学習を尊重 | 自律性支援（D-2） |
| **M_low** | Before/After Reading、学習方略ガイド、外的構造 | 暗黙知明示化（C-1）、外的構造（C-2） |

**Before Reading（M_low向け）**:
```
:::before-reading
## この記事を読む前に

**目的**: この記事を読むと、[達成目標] ができるようになります。

**前提知識**: [[前提概念A]], [[前提概念B]]

**読み方のヒント**: 
- Phase 3のLaboratoryで実際に操作してみてください
- 分からない用語は [[用語]] をクリックで確認できます
:::
```

**After Reading（M_low向け）**:
```
:::after-reading
## 学習を振り返る

**理解できたこと**:
- [ ] [チェック項目1]
- [ ] [チェック項目2]

**効果的な復習方法**:
- 3日後にQuizを再挑戦（間隔反復）
- 類似問題を解く（転移練習）
:::
```

### 2.4 P×T次元への適応

| P | T | 表示モード | Phase構造 |
| :--- | :--- | :--- | :--- |
| **P_cognitive** | **T_extended** | 学習モード | 全Phase表示 |
| **P_reception** | **T_quick** | 参照モード | TL;DR + 定義のみ |
| **P_production** | any | 産出支援モード | 用例、コロケーション中心 |
| **P_solving** | any | 問題解決モード | 手順、適用例中心 |

**参照モード（P_reception × T_quick）のレイアウト**:
```
┌─────────────────────────────────────┐
│ TL;DR（1-2文の要約）                 │
├─────────────────────────────────────┤
│ 定義                                 │
├─────────────────────────────────────┤
│ [詳しく学ぶ] ← P_cognitive に遷移    │
└─────────────────────────────────────┘
```

**学習モード（P_cognitive × T_extended）のレイアウト**:
```
┌─────────────────────────────────────┐
│ Before Reading（M_lowの場合）        │
├─────────────────────────────────────┤
│ ┌─ Core Layer ─────────────────┐    │
│ │ TL;DR、定義                  │    │
│ └───────────────────────────────┘    │
├─────────────────────────────────────┤
│ ┌─ Learning Layer ─────────────┐    │
│ │ Step 1: 先行知識活性化           │    │
│ │ Step 2: モデル更新           │    │
```
