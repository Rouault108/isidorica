# Isidorica プロジェクト理念・哲学

このドキュメントは、Isidoricaプロジェクトの根幹となる**理念（Why）**と**価値観（Values）**を定義するものです。
理論的基盤（教育的公正、情報正義等）は [CORE_FOUNDATIONS.md](./CORE_FOUNDATIONS.md) を、学習設計の理論は [LEARNING_FOUNDATIONS.md](./LEARNING_FOUNDATIONS.md) を参照してください。

---

## ミッション（使命）

> **「人類の知識を、すべての人が理解できる形で提供する」**

Wikipediaが「事実の羅列（辞書）」であるならば、Isidoricaは**「理解への導き（教科書）」**を目指す。
知を公共化すること。インターネット時代の公共図書館。

---

## ビジョン（目指す世界）

情報の断片化とAIによるハルシネーションが蔓延する時代において、**「確かな知識」**と**「学ぶ喜び」**を取り戻す。
学習者が最後に辿り着き、最初に戻ってくる場所。デジタル時代のアレクサンドリア図書館。

---

## 核心的な価値観 (Core Values)

### 1. 教育的公正 (Educational Equity)

| 原則 | 内容 |
|------|------|
| **アクセスの平等** | 知識へのアクセスは、経済的・文化的背景に関わらず平等であるべき |
| **暗黙知の明示化** | 「学び方」を知っている層だけが有利にならないよう、学習のルールを明示する |
| **構造の提供** | 「自由に探索して」ではなく、学習パスという外的構造を提供する |

> **理論的基盤**: Bourdieuの文化資本論、Bernsteinの言語コード論、苅谷の意欲格差論。詳細は [CORE_FOUNDATIONS.md §1](./CORE_FOUNDATIONS.md) を参照。

---

### 2. 信頼性 (Trustworthiness)

| 原則 | 内容 |
|------|------|
| **出典至上主義** | 事実の正確性は「信頼できる出典へのリンク」によってのみ担保する |
| **権威勾配** | フラットな相互監視（Wikipedia）と、強い権威による一方的発信（専門書）の中間を目指す |
| **AIの役割限定** | AIは「真実の決定者」ではなく「対話の支援者」として配置する。内容の正誤は判断せず、論証構造の明確化、対話の促進、建設的表現の提案を行う |

#### 権威の源泉についてのスタンス
Isidoricaにおける「権威」は、以下の源泉から構成される。

1.  **出典の質 (Source Quality)**: 記述の正確性は、リンクされた出典の信頼性によって担保される。これが最も強い権威であり、執筆者の身元に依存しない。出典の信頼度はティア制（後述）で明文化する。
2.  **貢献の実績 (Track Record)**: Isidorica内での過去の貢献履歴（誤情報の訂正、建設的なレビュー、精度の高い編集）に基づく信頼。正式版で導入予定のアカウントシステムにより管理される。
3.  **透明性 (Transparency)**: 全ての編集履歴とディスカッションを公開し、読者と他の編集者が判断できるようにする。

> **Beta版の制約**: 初期リリース（Beta版）ではアカウントシステムを持たず、編集権限は編集長および招待者に限定される。貢献履歴はGitHub上のコミット履歴で間接的に追跡する。

---

### 3. 学習体験 (Learning Experience)

| 原則 | 内容 |
|------|------|
| **外在的負荷の最小化** | 広告、過剰な装飾、遅い表示を排除する |
| **洗練されたミニマリズム** | 装飾ではなくコンテンツを主役にする |
| **パフォーマンス至上主義** | 「爆速の表示」自体をUXの価値とする |

#### 理論的基盤
Isidoricaの学習設計は、予測処理、メタ認知、自己決定理論、身体化認知など、複数の学術領域にまたがる理論に基づいている。
詳細は [LEARNING_FOUNDATIONS.md](./LEARNING_FOUNDATIONS.md) を参照。

---

### 4. 対象者適応 (Audience Adaptation)

| 原則 | 内容 |
|------|------|
| **嘘ではなくモデル** | 初心者向け説明は「教育的に有効なモデル」であり、嘘ではない |
| **単一ソース・多層表示** | 記事を分離せず、詳細度を可変にする |
| **段階的な厳密化** | 導入から厳密な定義へ、読者のスキーマに合わせて深化させる |

#### 理論的基盤：教訓的転置 (Didactic Transposition)
「対象者の現在のスキーマで理解可能な、限定されたモデル」を提供する。
例: 原子のボーア・モデル（厳密には誤りだが、学習初期には有用）。

## 3つの基盤 (Three Foundations)

Isidoricaは以下の3つの基盤から構成される。これは「知識の獲得」が多面的なプロセスであるという哲学に基づく。

| 基盤 | 哲学的根拠 | 役割 |
|:---|:---|:---|
| **Library** | 知識は体系的に構造化されなければ理解できない | 知識の体系的提供、学習パス |
| **Laboratory** | 知識は「知る」だけでなく「体験する」ことで身につく | 能動的推論、介入による学習 |
| **Agora** | 知識は社会的に構成され、批判的対話で精練される | 探究対話、知識の循環 |

> **理論的基盤**: Libraryは動的意味論（前提と文脈更新）、Laboratoryは身体化認知と能動的推論、Agoraは語用論とアーギュメンテーション理論に基づく。

## Isidorica の戦略的立ち位置

| 項目 | Wikipedia (現状) | Isidorica (目指す姿) |
| :--- | :--- | :--- |
| **信頼性の担保** | 衆人環視（誰でも編集可） | 権威勾配付きの衆人環視（Git + CODEOWNERS） |
| **出典の扱い** | 形式があればOK | 高信頼性の出典のみを許可 + 自動検証 |
| **記事の性格** | 辞書（事実の羅列） | 教科書（理解への導き） |
| **対象読者** | 「平均的な読者」という幻想 | 初学者から専門家まで（レイヤー化） |
| **UI/UX** | 2000年代のテキストサイト | 「理解」を加速させる機能美 |
| **コミュニティ** | 複雑なローカルルールと同調圧力 | 明確なCI/CDパイプラインとAIによる形式監査 |
