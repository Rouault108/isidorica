# レビュワーガイドライン (Reviewer Guidelines)

本ドキュメントでは、Isidoricaのレビュワーが記事・翻訳をレビューする際の基準と手順を定義する。

**ステータス**: ドラフト

---

## 1. レビュワーの役割

| 責務 | 説明 |
| :--- | :--- |
| **品質担保** | 記事の正確性、中立性、読みやすさを確認 |
| **出典確認** | 出典の適切性、信頼性を確認 |
| **翻訳関係設定** | 必要な場合に原著と翻訳版を紐付け |
| **翻訳品質確認** | 翻訳版の言語的正確性を確認 |

---

## 2. 新規記事のレビュー

### 2.1 チェックリスト

| カテゴリ | チェック項目 |
| :--- | :--- |
| **出典** | 全ての主張に出典があるか |
| **出典** | 出典は信頼できるか（学術論文、書籍、公的機関等） |
| **出典** | 単一出典への過度な依存がないか |
| **中立性** | 特定の立場に偏っていないか |
| **中立性** | 広告的な記述がないか |
| **正確性** | 事実誤認がないか |
| **読みやすさ** | 専門用語に適切な説明があるか |
| **構造** | 見出しが論理的に構成されているか |

### 2.2 CIレポートの確認

PRに自動生成されるCIレポートを確認する。

| レポート項目 | 対応 |
| :--- | :--- |
| **新規出典の検出** | 出典の信頼性を確認 |
| **ページ番号なし警告** | 文脈を確認し、必要なら追加を依頼 |
| **出典ID衝突** | 衝突を解決 |

### 2.3 翻訳関係の設定

新規出典に翻訳関係が必要か判断する。

| ケース | 対応 |
| :--- | :--- |
| **哲学書、古典** | 原著と各言語訳を紐付け |
| **学術論文** | 通常は不要（原著のみ） |
| **一般書籍** | 翻訳版が存在すれば紐付け |

---

## 3. 翻訳版のレビュー

### 3.1 レビュー要件

翻訳版をレビューする際、以下のいずれかの方法で品質を確認する。

| 方法 | 説明 | 適用 |
| :--- | :--- | :--- |
| **ネイティブレビュワー** | その言語のネイティブスピーカーがレビュー | 最も信頼性が高い |
| **言語堪能者** | ネイティブでなくても十分な言語力を持つ者がレビュー | 許容 |
| **AI判定** | AIが翻訳品質をチェック | 補助的使用、最終責任は人間 |

### 3.2 チェックリスト

| カテゴリ | チェック項目 |
| :--- | :--- |
| **一貫性** | オリジナル版と内容が一致しているか |
| **出典** | 出典リストがオリジナル版と一致しているか（CI自動チェック） |
| **用語** | 専門用語の訳が適切か |
| **用語** | 同じ概念に同じ訳語を使っているか |
| **自然さ** | 機械翻訳臭がなく、自然な表現か |
| **文法** | 文法的に正しいか |

### 3.3 機械翻訳について

*   **使用は許可**: 機械翻訳の使用を禁止も推奨もしない。
*   **開示義務なし**: 機械翻訳の使用有無を開示する義務はない。
*   **レビュー必須**: 機械翻訳を使用した場合も、レビューを経て公開する。

### 3.4 AI判定の使用

| 項目 | 説明 |
| :--- | :--- |
| **役割** | 補助的なチェック |
| **責任** | 最終的な承認責任は人間のレビュワー |
| **チェック項目** | 意味の一貫性、用語の統一、文法・自然さ |

## 4. 出典の評価基準

### 4.1 信頼性の高い出典

| カテゴリ | 例 |
| :--- | :--- |
| **学術論文** | 査読付きジャーナル |
| **書籍** | 学術出版社、専門出版社 |
| **公的機関** | 政府機関、国際機関の公式文書 |
| **百科事典** | 専門分野の百科事典（一般百科事典は補助的） |

### 4.2 注意が必要な出典

| カテゴリ | 対応 |
| :--- | :--- |
| **Wikipedia** | 二次情報源として不適切、ただし出典探しの入口として利用可 |
| **個人ブログ** | 原則不可、例外は当該分野の専門家のブログ |
| **ニュース記事** | 速報性の高い情報のみ、長期的な記述には不適切 |
| **SNS** | 原則不可 |

### 4.3 単一出典への依存

*   一つの記事が単一の出典にのみ依存している場合、追加出典を要求。
*   自著引用のみの場合、特に注意。

---

## 5. 中立性の確認

### 5.1 偏りの検出

| 偏りの種類 | 例 |
| :--- | :--- |
| **立場の偏り** | 特定の学派・思想のみを正当化 |
| **選択の偏り** | 都合の良い出典のみを引用 |
| **表現の偏り** | 誘導的な言い回し |
| **広告的偏り** | 特定の製品・サービスを過度に推奨 |

### 5.2 対応

*   偏りが見られる場合、修正を依頼。
*   複数の視点を公平に紹介するよう求める。

---

## 6. レビューのプロセス

### 6.1 基本フロー

```
PR作成
  ↓
CIレポート確認
  ↓
記事内容レビュー（チェックリスト）
  ↓
修正依頼（必要な場合）
  ↓
修正確認
  ↓
承認
  ↓
マージ
```

### 6.2 コメントの書き方

| 種類 | 例 |
| :--- | :--- |
| **必須修正** | 「この主張には出典が必要です。」 |
| **推奨修正** | 「より明確な表現に変更することを推奨します。」 |
| **質問** | 「この解釈の根拠を教えてください。」 |
| **称賛** | 「この説明は非常にわかりやすいです。」 |

---

## 7. レビュワーになるには

### 7.1 要件

| 要件 | 説明 |
| :--- | :--- |
| **貢献実績** | 一定数の記事執筆・翻訳の実績 |
| **品質** | 過去の貢献が高品質であること |
| **コミュニティ** | ディスカッションでの建設的な参加 |

### 7.2 プロセス

*   管理者による招待または自薦
*   既存レビュワーによる承認
