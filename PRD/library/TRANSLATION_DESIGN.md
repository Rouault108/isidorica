# 翻訳設計 (Translation Design)

Isidoricaにおける多言語対応の設計仕様。翻訳ワークフロー、品質基準、技術実装を定義する。

**関連ドキュメント**:
- [TRANSLATION_FOUNDATIONS.md](../shared/TRANSLATION_FOUNDATIONS.md): 翻訳の理論的基盤
- [GOVERNANCE.md](./GOVERNANCE.md): コンテンツガバナンス
- [USER_SCENARIOS.md](./USER_SCENARIOS.md): 翻訳者のユースケース（UC-T1〜T4）

## 概要

### 設計原則

| 原則 | 説明 | 理論的根拠 |
| :--- | :--- | :--- |
| **動的等価優先** | 読者の理解を最優先し、原文の効果を再現 | Nida (1964) |
| **スコポス準拠** | 翻訳の目的は「読者が概念を理解し、能力を獲得すること」 | Vermeer (1978) |
| **文化適応推奨** | 読者の文化圏に適した補足を追加 | 文化的等価 |
| **透明性** | 翻訳の判断・限界を必要に応じて注記 | 翻訳倫理 |
| **オリジナル尊重** | 翻訳版はオリジナルの忠実な反映 | 忠実性原則 |

## 1. 多言語コンテンツ構造

### 1.1 オリジナル言語の概念

記事ごとに「**オリジナル言語（原著言語）**」を Frontmatter で宣言する。

```yaml
slug: prisoner-dilemma
title: 囚人のジレンマ
original_language: ja  # この記事のオリジナル言語
```

| 概念 | 説明 |
| :--- | :--- |
| **オリジナル言語** | その記事が**最初に執筆された言語** |
| **翻訳版** | オリジナル言語以外の言語版 |
| **正本（Canonical）** | オリジナル言語版が正本。内容の変更はここで行う |

### 1.2 新規執筆

**任意の言語で新規執筆可能**。

| ルール | 説明 |
| :--- | :--- |
| **言語選択の自由** | 執筆者は自身の母語または最も得意な言語で執筆してよい |
| **自動決定** | 最初に書かれた言語が自動的にオリジナル言語となる |
| **混在の許容** | 日本語ベースの記事と英語ベースの記事が混在することを許容 |

### 1.3 翻訳版のルール

| ルール | 説明 |
| :--- | :--- |
| **オリジナル準拠** | 翻訳版はオリジナル言語版の内容に基づく |
| **出典の追加・削除禁止** | 翻訳者が独自に出典を追加・削除することは**禁止**。追加したい場合はオリジナル版に提案 |
| **補足説明の許可** | 翻訳者は補足説明（例: 「この出典は日本語のみ」）を追加可能 |

### 1.4 同期の原則

翻訳版とオリジナル版の関係を定義する。

| 原則 | 説明 |
| :--- | :--- |
| **翻訳はオリジナルの忠実な反映** | 翻訳版はオリジナルの内容を正確に伝えることが第一義 |
| **同期の方向** | オリジナル → 翻訳版（一方向）。翻訳版の独自変更がオリジナルに自動反映されることはない |
| **改善提案** | 翻訳中に発見した誤りや改善点は、オリジナルに対してIssue/PRを作成 |

> **将来の検討事項**: リポジトリ分割後は、クロスリポジトリIssue/PRの運用方法を検討する。

## 2. 翻訳ワークフロー

### 2.1 ワークフロー概要

詳細なワークフローは [USER_SCENARIOS.md](./USER_SCENARIOS.md) の翻訳者セクション（UC-T1〜T4）を参照。

```
翻訳対象記事の選択
    ↓
翻訳作業（新規ファイル作成）
    ↓
PR作成
    ↓
CIチェック（出典リストの一致確認）
    ↓
言語レビュー
    ↓
マージ → 公開
```

### 2.2 機械翻訳の使用

| ポリシー | 内容 |
| :--- | :--- |
| **許可** | 機械翻訳（DeepL、Google Translate、AI翻訳等）の使用は禁止も推奨もしない |
| **レビュー必須** | 機械翻訳を使用した場合も、レビューを経て公開する |
| **判別不要** | 機械翻訳の使用有無を開示する義務はない（判別困難なため） |

**理論的根拠**: [TRANSLATION_FOUNDATIONS.md](../shared/TRANSLATION_FOUNDATIONS.md) セクション5参照。機械翻訳は「下訳」として活用され、人間のポストエディットにより品質を確保する。

### 2.3 翻訳版のレビュー

| 方法 | 説明 |
| :--- | :--- |
| **ネイティブレビュワー** | その言語のネイティブスピーカーがレビュー |
| **言語堪能者** | ネイティブでなくても十分な言語力を持つ者がレビュー |
| **AI判定** | AIが翻訳品質をチェック（補助、最終責任は人間） |

詳細な判断基準は [REVIEWER_GUIDELINES.md](./REVIEWER_GUIDELINES.md) を参照。

### 2.4 Language Reviewer

特定言語の翻訳レビューを担当する役割。

| 項目 | 内容 |
| :--- | :--- |
| **選任基準** | その言語のネイティブまたは十分な言語力を持つ者 |
| **権限** | 該当言語の翻訳PRをApprove/Request Changes |
| **責任** | 翻訳品質の保証、用語の一貫性確認 |

## 3. 文化適応

### 3.1 文化適応とは

翻訳版において、読者の文化圏に適した補足を追加すること。

> **理論的根拠**: スコポス理論に基づき、翻訳の目的（読者の理解）に合わせて適応を行う。[TRANSLATION_FOUNDATIONS.md](../shared/TRANSLATION_FOUNDATIONS.md) セクション1.4参照。

### 3.2 推奨される文化適応

| 適応タイプ | 説明 | 例 |
| :--- | :--- | :--- |
| **文化固有の例示** | 読者の文化圏に馴染みのある例を追加 | 日本語版で日本の事例を追加 |
| **言語固有の説明** | その言語固有の文法説明等を追加 | 言語学習の文脈で |
| **慣用表現の対照** | 原語の慣用表現に対応する現地の表現を紹介 | 「Schadenfreude」→「人の不幸は蜜の味」 |

### 3.3 文化適応のルール

| ルール | 説明 |
| :--- | :--- |
| **オリジナルの保持** | 文化適応はオリジナルの内容を**置き換える**のではなく、**補足**する形で追加 |
| **注記の追加** | 適応した箇所には必要に応じて注記を付ける |
| **レビューで確認** | 文化適応の妥当性はレビューで確認 |

**推奨理由**: Isidoricaは公共知として、各文化圏の読者に最も理解しやすい形で知識を提供すべきである。文化適応は読者の認知的障壁を下げ、学習効果を高める。

## 4. 出典の翻訳関係

### 4.1 概要

原著と翻訳書の関係を管理し、読者の言語に応じて適切な版を表示する。

| ポイント | 説明 |
| :--- | :--- |
| **多くの出典は設定不要** | 翻訳関係の設定が必要なのは、原著と複数の翻訳版がある場合のみ（哲学書、古典等） |
| **自動選択** | ビルド時に読者の言語に応じて適切な出典を選択 |

### 4.2 ビルド時の自動処理

ビルド時にremarkプラグインが `bibliography.json` を読み込み、現在の言語版に応じて適切な出典を選択。

| 言語版 | 表示される出典 |
| :--- | :--- |
| **日本語版** | 日本語翻訳（存在する場合） |
| **英語版** | 英語翻訳または原著 |
| **原著言語版** | 原著情報 |

**追加ルール**:

*   原著情報も併記される（例: `（ハイデガー 2003, p.45; 原著: Heidegger 1927）`）
*   翻訳がない場合は、原著情報を出力し、言語注記を追加（例: `(In German)`）

### 4.3 翻訳関係の設定（レビュー時）

| ステップ | 説明 |
| :--- | :--- |
| **1. 判断** | PRレビュー時に、レビュワーが翻訳関係の設定が必要か判断 |
| **2. 設定** | 必要な場合は管理UIで原著・翻訳版を紐付け |
| **3. 反映** | 設定内容は `bibliography.json` に反映 |

### 4.4 手動入力資料の翻訳関係

国際条約、他国法令、古典文献など、公式翻訳や定訳が存在する資料は翻訳関係を設定可能。

| ケース | 翻訳関係設定 | 例 |
| :--- | :--- | :--- |
| 公式翻訳がある | 設定する | 国連憲章（英語原文 ↔ 日本語公定訳） |
| 学術的な定訳がある | 設定する | プラトン『国家』（ギリシャ語 ↔ 各国語訳） |
| 非公式翻訳のみ | 設定しない | 原典を使用、言語注記を追加 |
| 翻訳がない | 設定しない | 原典のみ表示 |

**運用**:

*   執筆者は**原典のID**を使用（例: `[@un-charter]`）
*   翻訳関係が設定されていれば、ビルド時に言語版に応じて自動選択
*   入力フォームで翻訳版を追加登録可能

### 4.5 翻訳版でのページ番号

翻訳者はページ番号を翻訳版に合わせて追加・変更することが許可される（レビューで確認）。

| ケース | 対応 |
| :--- | :--- |
| オリジナル版にページ番号がない | 翻訳版で追加可能 |
| オリジナル版と異なるページ番号 | 翻訳版の出典に合わせて変更可能 |
| 原典ページとの併記 | `[@id-ja, p. 45; orig. p. 42]` 形式で記載可能 |

**表示ロジック**

*   **基本**: 引用された版のページ番号をそのまま表示
*   **自動切替なし**: `[@id-ja, p. 45]` は日本語版p.45を固定表示（言語に応じて原典ページに自動変換しない）
*   **両方併記時**: 読者の言語に応じて主従を切り替え

| 引用 | 日本語版記事での表示例 | ドイツ語版（原典の言語）記事での表示例 |
| :--- | :--- | :--- |
| `[@id-ja, p. 45]` | ハイデガー 2003, p.45 | Heidegger (jap übers. 2003), p.45 |
| `[@id-ja, p. 45; orig. p. 42]` | ハイデガー 2003, p.45（原著 p.42） | Heidegger 1927, p.42 (jap übers. p.45) |

### 4.6 原典ページの併記

執筆者が翻訳版と原典のページ対応を確認した場合、両方を記載可能。

| 記法 | 意味 |
| :--- | :--- |
| `[@id-ja, p. 45; orig. p. 42]` | 日本語版p.45、原典p.42 |
| `[@id-ja, pp. 45-50; orig. pp. 40-45]` | ページ範囲も同様 |

### 4.7 自動選択ルール

| ケース | 処理 |
| :--- | :--- |
| `[@source-id]`（ページなし） | 言語版に応じて翻訳版を自動選択 |
| `[@source-id-ja, p. 45]`（ページあり） | 指定された版をそのまま使用（自動選択しない） |
| `[@source-id-ja, p. 45; orig. p. 42]`（両方） | 言語版に応じて主従を切り替え |

**表示例**

| 記法 | 日本語版での表示 | 英語版での表示 |
| :--- | :--- | :--- |
| `[@id-ja, p. 45]` | ハイデガー 2003, p.45 | Heidegger (ja trans. 2003), p.45 |
| `[@id, p. 42]` | Heidegger 1927, p.42 | Heidegger 1927, p.42 |
| `[@id-ja, p. 45; orig. p. 42]` | ハイデガー 2003, p.45（原著 p.42） | Heidegger 1927, p.42 (ja trans. p.45) |

### 4.8 ページ番号要件

| 引用タイプ | ページ番号 | CI検証 |
| :--- | :--- | :--- |
| **直接引用**（「」で囲む、blockquote） | **必須** | エラー（マージ不可） |
| **パラフレーズ**（特定箇所の言い換え） | **強く推奨** | 警告（レビュワーが判断） |
| **全体参照**（著作全体のアイデア） | 任意 | 警告なし |

## 5. ディレクトリ配置と改善提案

### 5.1 翻訳版のディレクトリ追従

翻訳版はオリジナル版と同じ主題ディレクトリに配置される。

> **詳細**: ディレクトリ追従の技術的ルールは [URL_AND_TAXONOMY_DESIGN.md](./technical/URL_AND_TAXONOMY_DESIGN.md) セクション4.5を参照。

### 5.2 ディレクトリ配置の改善提案

翻訳者がオリジナル記事の主題ディレクトリ（分野分類）に問題を発見した場合の対応プロセス。

**シナリオ例**:
- オリジナル執筆者が分野に迷い `misc/` に配置
- 翻訳者がその分野に詳しく、`economics/` が適切と判断

**対応プロセス**

| プロセス | 説明 | 適用ケース |
| :--- | :--- | :--- |
| **A. 2段階プロセス** | (1) オリジナルに対してIssue/PRで改善提案 → (2) オリジナル移動後、翻訳版の移動PRが自動作成される | 翻訳者がオリジナルを直接編集しない場合 |
| **B. 同時移動PR** | オリジナルと翻訳版を同時に移動するPRを作成 | 翻訳者がオリジナルも編集可能な場合（効率的） |

**推奨**: 可能な限り **B（同時移動PR）** を使用する。1回のPRで完了し、効率的。

> **注意**: ディレクトリを変更してもURLは変わらない（URLに主題ディレクトリは含まれない）。

## 6. 技術実装

### 6.1 ファイル構造

```
content/
└── articles/
    ├── ja/                    # 日本語版
    │   └── philosophy/
    │       └── prisoner-dilemma.md
    ├── en/                    # 英語版
    │   └── philosophy/
    │       └── prisoner-dilemma.md
    └── de/                    # ドイツ語版
        └── philosophy/
            └── prisoner-dilemma.md
```

### 6.2 Frontmatter

```yaml
# 日本語版（オリジナル）
slug: prisoner-dilemma
title: 囚人のジレンマ
original_language: ja

# 英語版（翻訳）
slug: prisoner-dilemma
title: Prisoner's Dilemma
original_language: ja  # オリジナル言語を参照
```

### 6.3 翻訳同期のチェック

CIで以下をチェック：

| チェック項目 | 説明 |
| :--- | :--- |
| **出典リストの一致** | 翻訳版の出典IDリストがオリジナルと一致するか |
| **構造の一致** | 見出し構造が大きく乖離していないか（警告レベル） |
| **オリジナルの更新検知** | オリジナルが更新された場合、翻訳版に通知 |

### 6.4 翻訳同期ダッシュボード

管理者向けダッシュボードで翻訳版の同期状態を表示。

| 指標 | 説明 |
| :--- | :--- |
| **翻訳同期率** | オリジナルと同期済み翻訳版の割合 |
| **未翻訳記事** | 翻訳版が存在しない記事のリスト |
| **更新待ち** | オリジナルが更新されたが翻訳版が未更新の記事 |

> **詳細**: [GOVERNANCE.md](./GOVERNANCE.md) セクション8.3参照。

## 7. 品質基準

### 7.1 翻訳品質の評価観点

| 観点 | 基準 | 重要度 |
| :--- | :--- | :--- |
| **正確性** | 概念が正確に伝わっているか | 最高 |
| **明瞭性** | 読者にとって理解しやすいか | 高 |
| **自然さ** | ターゲット言語として自然な表現か | 高 |
| **一貫性** | 用語・文体がサイト全体で統一されているか | 中 |
| **文化適応性** | 読者の文化圏に適した例示・説明があるか | 中 |
| **技術的正確さ** | Laboratory等のコードが正しく機能するか | 状況による |

> **理論的根拠**: [TRANSLATION_FOUNDATIONS.md](../shared/TRANSLATION_FOUNDATIONS.md) セクション6.3参照。

### 7.2 許容される翻訳判断

翻訳者が独自に判断してよい範囲：

| 許容 | 禁止 |
| :--- | :--- |
| 表現の言い換え（意味が同じ） | 内容の追加・削除 |
| 文化適応の追加（補足として） | 出典の追加・削除 |
| ページ番号の変更（翻訳版に合わせて） | オリジナルの意図に反する改変 |
| 補足説明の追加（注記として） | 構造の大幅な変更 |

### 7.3 エスカレーションが必要なケース

以下の場合は、オリジナルへのIssue/PR作成が必要：

| ケース | 対応 |
| :--- | :--- |
| オリジナルに誤りを発見 | Issue作成 |
| 内容の追加・改善が必要 | PR作成（オリジナルへ） |
| 出典の追加・削除が必要 | PR作成（オリジナルへ） |
| ディレクトリ配置の問題 | Issue/PR作成 |

## 8. 用語管理

### 8.1 用語集の必要性

一貫した翻訳のために、用語集（Glossary）を管理する。

| 目的 | 説明 |
| :--- | :--- |
| **一貫性** | 同じ概念は同じ訳語で |
| **品質向上** | 適切な訳語の選択を支援 |
| **効率化** | 翻訳者間での用語の共有 |

### 8.2 用語集の構造

```yaml
# glossary/ja-en.yml
terms:
  - source: "囚人のジレンマ"
    target: "Prisoner's Dilemma"
    domain: game-theory
    note: "正式な学術用語"
  
  - source: "ナッシュ均衡"
    target: "Nash Equilibrium"
    domain: game-theory
    alternative: ["Nash equilibria (複数形)"]
```

### 8.3 用語集の管理

| 項目 | 内容 |
| :--- | :--- |
| **追加** | 翻訳者がPRで提案、Language Reviewerが承認 |
| **変更** | 既存訳語の変更は慎重に（既存記事への影響を確認） |
| **ドメイン別** | 分野ごとに用語を整理 |

### 8.4 CAT（翻訳支援ツール）との連携（将来）

将来的に、翻訳支援ツール（CAT: Computer-Assisted Translation）との連携を検討。

| 機能 | 説明 |
| :--- | :--- |
| **用語チェック** | 用語集との整合性を自動チェック |
| **翻訳メモリ** | 過去の翻訳を再利用 |
| **一括置換** | 用語変更時に全記事を一括更新 |

## 9. Laboratory の多言語対応

### 9.1 概要

Laboratoryコンポーネントは記事本文とは異なる多言語対応が必要。

| 要素 | 多言語対応方法 |
| :--- | :--- |
| **コード（LAB-CODE）** | 言語非依存（翻訳不要） |
| **コード内コメント** | 翻訳対象（オプション） |
| **UI要素（ボタン等）** | 翻訳対象 |
| **説明文** | 記事本文として翻訳 |
| **エラーメッセージ** | 翻訳対象（将来） |

### 9.2 コード内コメントの翻訳

```python
# 日本語版
# ナッシュ均衡を計算する
def nash_equilibrium(matrix):
    ...

# 英語版
# Calculate Nash Equilibrium
def nash_equilibrium(matrix):
    ...
```

| 方針 | 説明 |
| :--- | :--- |
| **翻訳推奨** | コメントは学習者の理解を助けるため、翻訳を推奨 |
| **変数名は維持** | 変数名・関数名は英語のまま維持（プログラミング慣習） |

### 9.3 UI要素の翻訳

Laboratory共通のUI要素（「実行」ボタン等）はシステム側で多言語対応。

| 要素 | 日本語 | 英語 |
| :--- | :--- | :--- |
| 実行ボタン | 実行 | Run |
| リセットボタン | リセット | Reset |
| 出力エリア | 出力 | Output |
| エラー表示 | エラー | Error |

> **詳細**: [LABORATORY_DESIGN.md](../engine/LABORATORY_DESIGN.md) を参照。

## 10. 翻訳者向けガイドライン概要

詳細は [REVIEWER_GUIDELINES.md](./REVIEWER_GUIDELINES.md) および [CONTRIBUTOR_GUIDE.md](./CONTRIBUTOR_GUIDE.md) を参照。

### 10.1 翻訳者の心得

| 原則 | 説明 |
| :--- | :--- |
| **忠実性** | オリジナルの意味を正確に伝える |
| **誠実性** | 翻訳の限界を認め、必要に応じて注記 |
| **尊重** | 原著者の意図を尊重し、勝手な改変をしない |
| **改善提案** | 誤りを発見した場合はオリジナルに提案 |

> **理論的根拠**: [TRANSLATION_FOUNDATIONS.md](../shared/TRANSLATION_FOUNDATIONS.md) セクション6.2参照。

### 10.2 翻訳チェックリスト

- [ ] オリジナルの内容を正確に反映しているか
- [ ] ターゲット言語として自然な表現か
- [ ] 用語が用語集と一致しているか
- [ ] 出典リストがオリジナルと一致しているか
- [ ] 文化適応がオリジナルを補足する形になっているか
- [ ] Laboratoryのコメントが翻訳されているか（推奨）
