## 1. 概要

### 1.1 記事ページの役割

Isidoricaの**最も重要なページタイプ**。読者が知識を獲得する場所。

| 役割 | 説明 |
| :--- | :--- |
| **知識の提供** | 概念を体系的に説明 |
| **学習体験** | Phase 1-6の構造に沿った学習 |
| **信頼性の担保** | 出典へのアクセス、引用機能 |
| **探索の起点** | 関連記事、トピックへのナビゲーション |

### 1.2 URL構造

```
/{lang}/{slug}
```

例: `/ja/game-theory`

> **参照**: URL構造の詳細は [URL_AND_TAXONOMY_DESIGN.md](./technical/URL_AND_TAXONOMY_DESIGN.md) を参照。

## 3. 左サイドバー（学習パスナビ）

左サイドバーは「学習パスナビゲーション」を提供する。読者モードに応じて表示/非表示を切り替える。

### 3.1 表示条件

| 読者モード | 左サイドバー | 理由 |
| :--- | :--- | :--- |
| **Learner** | ✅ 表示 | 体系的学習を支援。「今どこにいるか」が明確 |
| **Explorer** | ❌ 非表示 | 概要を知りたい層。体系的学習は意図していない |
| **Scholar** | ❌ 非表示 | 既に理解している層。学習パスは不要 |

### 3.2 表示内容（Learnerモード）

| ケース | 表示内容 |
| :--- | :--- |
| **学習パスコンテキストあり** | 現在のパス内記事一覧（現在位置をハイライト） |
| **直接アクセス（記事が1つのパスに属する）** | そのパスを表示 |
| **直接アクセス（記事が複数パスに属する）** | `learning-paths.yml` で最初に定義されているパスをデフォルト表示 + 「他のパスで学ぶ」リンクで切り替え可能 |
| **直接アクセス（記事がどのパスにも属さない）** | 「関連する学習パス」を提案、または非表示 |

### 3.3 構成要素

| 要素 | 内容 |
| :--- | :--- |
| **パス名** | 「経済学入門パス」等 |
| **記事リスト** | パス内の全記事を順番に表示。現在の記事をハイライト |
| **進捗バー** | 「3/10 完了」等（将来機能） |
| **他のパスリンク** | 記事が複数パスに属する場合の切り替え |

### 3.4 ワイヤーフレーム

```
経済学入門パス
────────────────
  ① 需要と供給    ✔
  ② 市場均衡      ✔
▶ ③ ゲーム理論    ← 現在
  ④ ナッシュ均衡
  ⑤ 完全情報ゲーム
  ⑥ 外部性
────────────────
3/6 完了 [■■■□□□]

🔗 他のパスで学ぶ
```

### 3.5 「他のパスで学ぶ」ポップオーバー

記事が複数の学習パスに属する場合、別のパスに切り替える機能を提供する。

| 項目 | 仕様 |
| :--- | :--- |
| **トリガー** | 「🔗 他のパスで学ぶ」リンクをクリック |
| **表示形式** | ポップオーバー（サイドバーから右にはみ出す） |
| **表示内容** | パス名 + 説明 + 難易度（beginner/intermediate/advanced） |
| **選択時** | サイドバーが選択したパスに切り替わる |
| **保存** | 選択したパスはlocalStorageに保存（次回以降も適用） |

#### ワイヤーフレーム

```
🔗 他のパスで学ぶ ───────────────────────────┐
                 ┌───────────────────────────┐
                 │ この記事を含むパス          │
                 │ ─────────────────────────│
                 │ ● 経済学入門パス     [入門] │ ← 現在選択中
                 │   ゲーム理論の基礎概念を学ぶ │
                 │ ─────────────────────────│
                 │   高度なゲーム理論パス [応用] │
                 │   ナッシュ均衡の数学的証明   │
                 │ ─────────────────────────│
                 │   行動経済学パス     [中級] │
                 │   行動経済学の視点から学ぶ  │
                 └───────────────────────────┘
```

#### アクセシビリティ

| 要素 | 要件 |
| :--- | :--- |
| **ポップオーバー** | `role="dialog"` または `role="listbox"` |
| **現在選択中** | `aria-selected="true"` |
| **キーボード操作** | 矢印キーで選択、Enterで決定、Escで閉じる |
| **フォーカス管理** | 閉じた時に元のリンクにフォーカスを戻す |

### 3.6 アクセシビリティ（サイドバー全体）

| 要素 | 要件 |
| :--- | :--- |
| **ランドマーク** | `<nav aria-label="学習パス">` |
| **現在位置** | `aria-current="page"` |
| **完了状態** | `aria-label="完了済み"` またはチェックマークの代替テキスト |
| **進捗バー** | `role="progressbar"` + `aria-valuenow` |

### 3.7 パス切り替え時の連動更新

「他のパスで学ぶ」から別のパスを選択した場合、以下の箇所が**即座に連動して更新**される（ページ遷移なし）。

| 箇所 | 更新内容 |
| :--- | :--- |
| **左サイドバー（デスクトップ）** | 選択したパスの記事リストに切り替わる |
| **記事上部の学習パスナビ** | 選択したパスの「← 前 \| 次 →」に切り替わる |
| **記事下部の学習パスナビ** | 選択したパスの「← 前 \| 次 →」に切り替わる |
| **モバイルの簡易版パスナビ** | 選択したパス名・全体感・前後リンクに切り替わる |
| **localStorage** | 選択したパスIDを保存（次回以降も適用） |

## 5. 本文エリア

記事の主要コンテンツ。

### 5.1 Phase構造

記事はPhase 1-6の構造に従う（詳細は READER_ADAPTATION_DESIGN.md 参照）。

| Phase | 見出し | 目的 |
| :--- | :--- | :--- |
| **Phase 1** | 予測誤差の喚起 | 読者の予測を覆す |
| **Phase 2** | 定義 | 概念の正式な定義 |
| **Phase 3** | 機序 | 動作原理の説明 |
| **Phase 4** | 確認問題 | 理解度チェック |
| **Phase 5** | 文脈・歴史 | 歴史的・社会的意義 |
| **Phase 6** | 応用・転移 | 異なる文脈への転移 |

### 5.2 インラインコンポーネント

| コンポーネント | 用途 | 詳細 |
| :--- | :--- | :--- |
| **脚注（Footnote）** | 補足情報 | ポップオーバー形式 |
| **用語定義（Toggletip）** | 専門用語の説明 | `terms.yml` から取得 |
| **出典引用** | 参考文献への参照 | `[@id]` 形式、**ポップオーバー対応** |
| **コードブロック** | コード表示 | シンタックスハイライト、コピー機能 |
| **Laboratory** | インタラクティブ | WASMベースのシミュレーション |

### 5.3 出典引用のポップオーバー

| 項目 | 仕様 |
| :--- | :--- |
| **トリガー** | クリック（タップ） |
| **表示内容** | 著者、タイトル、出版年、出版社等 |
| **フォールバック** | 記事下部の参考文献リストへのリンク |
| **対応引用形式** | 全形式（Harvard, APA, Vancouver, IEEE等） |

#### アクセシビリティ要件

| 要件 | 対応 |
| :--- | :--- |
| **キーボード操作** | Enter/Spaceで開閉、Escで閉じる |
| **スクリーンリーダー** | `aria-expanded`、`aria-describedby`で状態をアナウンス |
| **フォーカス管理** | 閉じた時に元のリンクにフォーカスを戻す |

> **理由**: 番号形式（Vancouver, IEEE等）では `[1]` だけでは誰の研究か分からない。ポップオーバーで即座に詳細を確認できる。

### 5.4 ブロックコンポーネント

| コンポーネント | 用途 | Directive |
| :--- | :--- | :--- |
| **Callout** | 注意、警告、ヒント | `:::note`, `:::warning` |
| **Quiz** | 確認問題 | `:::quiz` |
| **Tabs** | コンテンツの切り替え | `:::tabs` |
| **Steps** | 手順の表示 | `:::steps` |

## 6. 右サイドバー（目次）

### 6.1 機能

| 機能 | 説明 |
| :--- | :--- |
| **見出し一覧** | Phase 1-6 + サブ見出し |
| **スクロール連動** | 現在表示中の見出しをハイライト |
| **クリックジャンプ** | クリックで該当見出しへスクロール |
| **スティッキー** | スクロール時に固定表示 |

### 6.2 表示項目

```
目次
├─ Phase 1: 予測誤差の喚起
├─ Phase 2: 定義
│   └─ 2.1 数学的定義
├─ Phase 3: 機序
│   ├─ 3.1 ナッシュ均衡
│   └─ 3.2 パレート最適
├─ Phase 4: 確認問題
├─ Phase 5: 文脈・歴史
├─ Phase 6: 応用・転移
└─ 参考文献
```

### 6.3 折りたたみ

*   デフォルト: Phase 見出しのみ表示
*   展開時: サブ見出しも表示
*   長い記事: 自動折りたたみ検討

## 7. フッターエリア

本文下部の領域。

### 7.1 構成要素

| 要素 | 位置 | 内容 |
| :--- | :--- | :--- |
| **参考文献リスト** | 本文直後 | 出典の一覧（ポップオーバーのフォールバック） |
| **関連記事** | 参考文献の下 | `concept-relations.yml` + トピック共通度 |
| **ディスカッションリンク** | 関連記事の下 | Agoraへのリンク（存在時） |
| **引用ボタン** | 最下部 | Permanent URL をコピー |
| **全文コピーボタン** | 最下部（引用ボタンの隣） | 記事全文をMarkdown形式でコピー |
| **学習パスナビ**（条件付） | 最下部 | 「← 前の記事 \| 次の記事 →」 |

### 7.2 引用ボタン

| 項目 | 仕様 |
| :--- | :--- |
| **ラベル** | 「引用する」または「Cite」 |
| **クリック時** | モーダル/ポップオーバーでPermanent URLを表示 |
| **表示内容** | Permanent URL（主）、Canonical URL（副） |
| **コピー機能** | ワンクリックでクリップボードにコピー |

### 7.3 全文コピーボタン

| 項目 | 仕様 |
| :--- | :--- |
| **ラベル** | 「📋 記事をコピー」または「📋 Markdownでコピー」 |
| **クリック時** | 記事全文をMarkdown形式でクリップボードにコピー |
| **コピー内容** | 記事本文（メタ情報、参考文献含む） |

> **参照**: 引用導線の詳細は [URL_AND_TAXONOMY_DESIGN.md](./technical/URL_AND_TAXONOMY_DESIGN.md) セクション2.6を参照。

## 8. 読者モード対応

### 8.1 モード切替UI

| 項目 | 仕様 |
| :--- | :--- |
| **位置** | フッターエリア（学習パスナビの近く） |
| **形式** | ドロップダウン |
| **デフォルト表示** | `📚 Learner ▼` |
| **デフォルト値** | Learner |
| **保存** | localStorage |

### 8.2 配置の理由

| 理由 | 説明 |
| :--- | :--- |
| **認知負荷の軽減** | 読み始める前に選択を強制しない |
| **自然な動線** | 読み進めて「変えたい」と思った時に切り替え |
| **localStorage保存** | 次回以降は設定済み |
| **デスクトップ・モバイル共通** | 同じ位置で一貫性 |

### 8.3 UIワイヤーフレーム

```
フッターエリア
┌───────────────────────────────────────────────────┐
│ 参考文献リスト                                      │
│ 関連記事                                           │
│ ───────────────────────────────────────────────── │
│ 📖 経済学入門パス（3/10）                            │
│   ← ナッシュ均衡 | 外部性 →                          │
│ ───────────────────────────────────────────────── │
│ 📚 Learner ▼   [引用する] [📋 記事をコピー]           │
└───────────────────────────────────────────────────┘
                ↑ 読者モード切替

クリック時:
┌─────────────────────┐
│ 📚 Learner ▼        │
│ ┌─────────────────┐ │
│ │ 🔍 Explorer     │ │
│ │ 📚 Learner ✓    │ │
│ │ 🎓 Scholar      │ │
│ └─────────────────┘ │
└─────────────────────┘
```

### 8.4 アクセシビリティ

| 要素 | 要件 |
| :--- | :--- |
| **ドロップダウン** | `<select>` または `role="listbox"` |
| **現在の選択** | `aria-selected="true"` |
| **キーボード操作** | 矢印キーで選択、Enterで決定 |

### 8.5 モードによる表示変化

| 要素 | Explorer | Learner | Scholar |
| :--- | :--- | :--- | :--- |
| **Phase内容量** | 圧縮 | フル | フル + α |
| **Laboratory** | ロック表示 | 利用可能 | 利用可能 |
| **出典** | 非表示 | インライン | インライン + 詳細 |
| **Quiz** | 簡易版 | フル | フル |

> **参照**: 読者モードの詳細は [READER_ADAPTATION_DESIGN.md](../engine/READER_ADAPTATION_DESIGN.md) を参照。

## 10. モバイル対応

### 10.1 レイアウト変更

| デスクトップ | モバイル |
| :--- | :--- |
| 3カラム | 1カラム |
| 読者モード切替（フッター） | フッター（同じ） |
| 左サイドバー（学習パスナビ） | 記事上部（パンくず下）+ 記事下部（フッター） |
| 右サイドバー（目次） | スティッキー目次バー + アコーディオン展開 |

### 10.2 学習パスナビのモバイル対応（Learnerのみ）

| 項目 | 仕様 |
| :--- | :--- |
| **上部（パンくず下）** | パス名 + 全体感（例: 3/10）+ 「← 前 \| 次 →」リンク |
| **下部（フッター）** | パス名 + 全体感（例: 3/10）+ 「← 前 \| 次 →」リンク |
| **表示内容** | 簡易版（リスト全体は見せない）+ 全体感 |
| **タップで展開** | ボトムシートでパス全体（記事リスト + 他のパスで学ぶ）を表示 |

```
       ↑ ボトムシート
```

### 10.3 目次のモバイル対応

**方針**: グローバルヘッダーの下にスティッキーな目次バーを配置。タップでアコーディオン展開。**目次のみ**を表示。

| 要素 | 仕様 |
| :--- | :--- |
| **グローバルヘッダー** | スティッキー（~50px） |
| **目次バー** | スティッキー、アコーディオン式（~40px） |
| **合計占有高さ** | ~90px（折りたたみ時） |
| **タップ時** | アコーディオンが展開し、**目次のみ**を表示 |

### 10.5 プログレスインジケーター

| 種類 | 採用 | 理由 |
| :--- | :--- | :--- |
| **スクロール進捗（円形/線形）** | ❌ 不使用 | 「読み終える」ことへの圧力になりうる。Isidoricaの目的は「読み終える」ではなく「理解する」こと |
| **現在のPhase表示** | ✅ 採用 | Phase構造はIsidoricaの核心。今どの学習段階かを知ることは有用 |

**目次バーの表示**:
*   折りたたみ時: 「▶ 目次」または「▶ Phase 3: 機序」（現在のPhase）
*   展開時: 全Phaseを表示、現在位置をハイライト

> **理論的根拠**: LEARNING_FOUNDATIONS.mdの「非競争的対話」に基づき、達成の数値化・プレッシャーを避ける。

## 13. コピー対応

### 13.1 基本方針

| 項目 | 方針 |
| :--- | :--- |
| **テキストコピー** | 制限なし。自由にコピー可能 |
| **引用情報の自動付加** | ❌ 行わない |
| **画像コピー** | 制限なし。透かしなし |

### 13.2 引用情報の自動付加を行わない理由

| 理由 | 説明 |
| :--- | :--- |
| **公共知としての本質** | Isidoricaの目的は「知識を最高の形で提供する」こと。オリジナル誘導は本質ではない |
| **ユーザー体験** | 自動付加はコピー体験を損なう可能性がある |
| **CC BY-SA 4.0の義務所在** | 帰属表示義務はコピーする側にあり、Isidoricaが強制するものではない |

### 13.3 引用支援機能

自動付加は行わないが、引用を**容易にする機能**は提供する。

| 機能 | 説明 |
| :--- | :--- |
| **引用ボタン** | Permanent URLをワンクリックでコピー（フッターエリアに配置） |
| **引用形式の生成** | APA、MLA、BibTeX等の形式で引用文を生成（将来機能） |

### 13.4 全文コピーボタン

| 項目 | 仕様 |
| :--- | :--- |
| **配置場所** | フッターエリア（引用ボタンの隣） |
| **形式** | Markdown形式（Isidoricaのソースと同じ） |
| **ラベル** | 「📋 記事をコピー」または「📋 Markdownでコピー」 |
| **コピー内容** | 記事本文（メタ情報、参考文献含む） |

| 理由 | 説明 |
| :--- | :--- |
| **公共知の流通** | ワンクリックでコピー → 知識の再利用がさらに容易 |
| **ツール連携** | Markdown形式でObsidian/Notion等との連携が容易 |
| **理念との整合性** | 知識を自由に流通させるIsidoricaの理念に完全に合致 |

> **理論的根拠**: 「強制」ではなく「支援」。公共知としての理念に合致。
