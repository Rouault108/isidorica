# フォーム設計 (Forms Design)

本ドキュメントでは、Isidoricaで使用するフォームの設計を定義する。

**ステータス**: ドラフト（Phase 4以降で実装予定）

---

## 1. 報告フォーム (Report Form)

GitHub Issueを作成できないユーザー向けの報告フォーム。

### 1.1 目的

*   出典の誤り、誤字脱字、コンテンツの問題を報告
*   GitHub APIを通じて自動的にIssueを作成

### 1.2 フィールド

| フィールド | 必須 | タイプ | 説明 |
| :--- | :--- | :--- | :--- |
| **報告種別** | ✓ | select | 出典の誤り / 誤字脱字 / 翻訳の問題 / 内容の問題 / **盗作・剽窃** / その他 |
| **記事URL** | ✓ | url | 問題のある記事のURL |
| **報告内容** | ✓ | textarea | 問題の詳細説明 |
| **報告者名** | - | text | 任意（匿名可） |
| **メールアドレス** | - | email | 任意（フォローアップ用） |

#### 盗作・剽窃報告の追加フィールド

報告種別で「盗作・剽窃」を選択した場合、以下のフィールドが追加表示される。

| フィールド | 必須 | タイプ | 説明 |
| :--- | :--- | :--- | :--- |
| **侵害URL** | ✓ | url | 侵害が見られる外部ページのURL |
| **IsidoricaのURL** | ✓ | url | 元のIsidorica記事のURL |
| **侵害の種類** | ✓ | select | 帰属表示なし / ライセンス違反 / 自作偽装 / その他 |
| **詳細説明** | - | textarea | 追加のコンテキスト |
| **連絡先メール** | - | email | 対応結果の通知希望時のみ |

> **参照**: 盗作・剽窃への対応フローについては [LEGAL_REQUIREMENTS.md](./LEGAL_REQUIREMENTS.md) のセクション6を参照。

### 1.3 ワークフロー

```
ユーザー: フォーム入力 → 送信
    ↓
hCaptcha: 検証（hCaptchaだけでは問題。あとで追記）
    ↓
Cloudflare Workers: データ受信
    ↓
GitHub API: Issue作成（isidorica-botアカウント）
    ↓
ユーザー: 完了メッセージ表示
```

### 1.4 Issue テンプレート

```markdown
## 報告種別
出典の誤り

## 報告対象
https://isidorica.org/ja/prisoner-dilemma

## 報告内容
ハイデガー『存在と時間』の引用でページ番号が間違っているようです。
p.45ではなくp.54ではないでしょうか。

## 報告者
匿名

---
*この Issue は Isidorica 報告フォームから自動作成されました。*
```

### 1.5 スパム対策

| 対策 | 説明 |
| :--- | :--- |
| **hCaptcha** | プライバシー重視のCAPTCHA（reCAPTCHAは使用しない） |
| **レート制限** | 同一IPからの連続投稿を制限（例: 5回/時間） |
| **ハニーポット** | 非表示フィールドへの入力を検出 |

### 1.6 技術スタック

| コンポーネント | 技術 |
| :--- | :--- |
| フロントエンド | React（Isidoricaサイト内） |
| バックエンド | Cloudflare Workers |
| CAPTCHA | hCaptcha |
| Issue作成 | GitHub API（Personal Access Token） |
| ボットアカウント | `isidorica-bot` |

## 2. 出典手動入力フォーム (Source Manual Entry Form)

外部APIで取得できない資料を手動登録するフォーム。

### 2.1 目的

*   Webページ、個人所蔵資料、法令、判例等の登録
*   執筆支援ツールの一部として提供

### 2.2 フィールド（共通）

| フィールド | 必須 | タイプ | 説明 |
| :--- | :--- | :--- | :--- |
| **資料タイプ** | ✓ | select | Webページ / 書籍 / 法令 / 判例 / 個人資料 / データセット |
| **タイトル** | ✓ | text | 資料のタイトル |
| **言語** | ✓ | select | 資料の言語 |

### 2.3 フィールド（資料タイプ別）

#### Webページ

| フィールド | 必須 | タイプ |
| :--- | :--- | :--- |
| URL | ✓ | url |
| アクセス日 | ✓ | date（自動入力） |
| 著者 | - | text |
| 公開日 | - | date |

#### 法令

| フィールド | 必須 | タイプ |
| :--- | :--- | :--- |
| 法律名 | ✓ | text |
| 法律番号 | - | text |
| 施行日 | - | date |
| 国 | ✓ | select |

#### 判例

| フィールド | 必須 | タイプ |
| :--- | :--- | :--- |
| 事件名 | ✓ | text |
| 裁判所 | ✓ | text |
| 判決日 | ✓ | date |
| 事件番号 | - | text |
| 出典 | - | text |

#### 個人所蔵・アーカイブ資料

| フィールド | 必須 | タイプ |
| :--- | :--- | :--- |
| 資料名 | ✓ | text |
| 所蔵者/所蔵機関 | ✓ | text |
| 参照番号 | - | text |
| 年代 | - | text |
| 所在地 | - | text |

### 2.4 翻訳版の追加

*   フォーム下部に「翻訳版を追加」ボタン
*   クリックで翻訳版フィールドが展開
*   複数の翻訳版を追加可能

## 3. お問い合わせフォーム (Contact Form)

一般的な問い合わせ用フォーム。

### 3.1 フィールド

| フィールド | 必須 | タイプ |
| :--- | :--- | :--- |
| お名前 | ✓ | text |
| メールアドレス | ✓ | email |
| 件名 | ✓ | text |
| 内容 | ✓ | textarea |

### 3.2 送信先

*   管理者メールアドレス（Cloudflare Email Routing）
*   または GitHub Discussions への投稿

---

## 4. アクセシビリティ要件

全てのフォームは以下を満たすこと：

| 要件 | 説明 |
| :--- | :--- |
| **ラベル** | 全フィールドに明示的なラベル |
| **エラー表示** | インラインエラー、スクリーンリーダー対応 |
| **キーボード操作** | Tab/Enterで操作可能 |
| **フォーカス** | 可視フォーカスインジケータ |
| **色のみに依存しない** | エラー表示はアイコンとテキストを併用 |
