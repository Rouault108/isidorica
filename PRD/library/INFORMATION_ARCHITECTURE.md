## 1. サイト構造 (Site Map)

```
isidorica.org/
│
├── /                           # ホームページ
│
├── /{lang}/                    # 言語別ルート
│   │
│   ├── {slug}                  # 📄 記事ページ（メインコンテンツ）
│   │
│   ├── paths/                  # 🎓 学習パス
│   │   ├── (index)             #   一覧ページ
│   │   └── {path-id}           #   個別パス詳細
│   │
│   ├── topics/                 # 🏷️ トピック
│   │   ├── (index)             #   一覧ページ
│   │   └── {topic-id}          #   個別トピックハブ
│   │
│   ├── archives/               # 📦 アーカイブ（過去版）
│   │   └── {slug}/{hash}       #   特定バージョン
│   │
│   ├── search/                 # 🔍 検索ページ
│   │
│   └── about/                  # ℹ️ About系ページ
│       ├── (index)             #   概要
│       ├── philosophy          #   理念・哲学
│       ├── contribute          #   貢献方法
│       └── license             #   ライセンス
│
├── /agora/                     # 💬 Agora（言語共通、将来機能）
│   ├── (index)                 #   全体の議論一覧
│   └── {slug}                  #   記事別ディスカッション
│
└── (静的ファイル)
    ├── robots.txt
    ├── sitemap.xml
    └── manifest.json
```

## 2. ページタイプ

| ページタイプ | URL | 目的 | 実装フェーズ |
| :--- | :--- | :--- | :--- |
| **ホームページ** | `/` | サービス紹介、Featured記事 | Phase 4 |
| **記事ページ** | `/{lang}/{slug}` | メインコンテンツ（知識の提供） | Phase 4 |
| **学習パス一覧** | `/{lang}/paths` | 学習パスの一覧と検索 | Phase 4 |
| **学習パス詳細** | `/{lang}/paths/{id}` | 個別パスの概要・記事リスト・進捗 | Phase 4 |
| **トピック一覧** | `/{lang}/topics` | トピックの一覧と検索 | Phase 4 |
| **トピック詳細** | `/{lang}/topics/{id}` | ナビゲーションハブ（関連記事・トピック） | Phase 4 |
| **Agora一覧** | `/agora` | 全ディスカッション一覧（言語共通） | Phase 6以降 |
| **記事別Agora** | `/agora/{slug}` | 記事に紐づく議論（言語共通） | Phase 6以降 |
| **アーカイブ** | `/{lang}/archives/{slug}/{hash}` | 記事の過去バージョン | Phase 4 |
| **検索** | `/{lang}/search` | 全文検索・フィルタリング | Phase 4 |
| **About** | `/{lang}/about/*` | サービス情報 | Phase 4 |

> **注意**: Agoraのみ言語プレフィックスなしの共通URL。詳細は [DISCUSSIONS_DESIGN.md](./DISCUSSIONS_DESIGN.md) セクション「多言語対応戦略」を参照。

#### 3.2.1 多言語ナビゲーション設計 (Linguistic Landscape)

**理論的背景**

**Landry & Bourhis (1997)** の言語景観論（Linguistic Landscape）は、公共空間における言語の可視性と言語の地位・活力の関係を分析する。Isidoricaはこれを多言語Web UIに適用し、言語の平等な扱いを設計原則とする。

**設計原則**

| 原則 | 内容 | 実装 |
| :--- | :--- | :--- |
| **言語の平等な可視性** | すべての対応言語を等しく表示 | 言語ドロップダウンに序列を設けない（アルファベット順） |
| **現在言語の明示** | 現在表示中の言語をUIで明確に示す | 言語切替ボタンに現在言語を表示（「日本語」等） |
| **自己表記原則** | 各言語をその言語自身で表記 | "日本語", "English", "한국어" 等 |
| **言語固有URL** | 言語ごとに異なるURL構造 | `/{lang}/` プレフィックス（`/ja/`, `/en/` 等） |

**言語切替UIの詳細仕様**

| 項目 | 仕様 |
| :--- | :--- |
| **表示形式** | 🌐 アイコン + 現在言語名（例: 🌐 日本語 ▼） |
| **ドロップダウン内容** | 対応言語を自己表記で一覧表示 |
| **言語順序** | ISO 639-1コードのアルファベット順（en → ja → ko...） |
| **未翻訳表示** | 翻訳がない言語は淡色表示 + "(coming soon)" |

**フォールバック戦略**

| シナリオ | 対応 |
| :--- | :--- |
| **記事が未翻訳** | 言語切替で他言語を選択すると、その言語のホームページにリダイレクト |
| **部分翻訳** | 翻訳済み部分を表示し、未翻訳部分は原語で表示 + 注記 |
| **言語自動検出** | `Accept-Language` ヘッダーで初回訪問時の言語を推定（ユーザー選択を優先） |

## 4. 検索とディスカバリー

> **ステータス**: 🔄 ドラフト

### 4.1 コンテンツ発見の方法

ユーザーがコンテンツを発見する経路は以下の通り：

| 発見方法 | 説明 | 対応ページ/機能 |
| :--- | :--- | :--- |
| **検索** | キーワードで記事を検索 | 検索ページ、ヘッダー検索 |
| **学習パス** | 体系的な学習ルート | 学習パス一覧・詳細 |
| **トピック** | 概念ベースのナビゲーション | トピック一覧・詳細 |
| **関連記事** | 現在の記事から派生 | 記事ページ下部 |
| **外部流入** | 検索エンジン、SNS | OGP、SEO |

### 4.2 検索機能

| 項目 | 仕様 |
| :--- | :--- |
| **技術** | Algolia DocSearch（主）/ Pagefind（フォールバック） |
| **検索対象** | 記事タイトル、本文、トピック、用語定義 |
| **フィルター** | トピック、言語、学習パス |
| **検索UI** | ヘッダーにアイコン → モーダル表示 |
| **検索ページ** | `/{lang}/search` で詳細検索・結果一覧 |

### 4.3 ディスカバリー機能

| 機能 | 配置 | 内容 |
| :--- | :--- | :--- |
| **Featured Articles** | ホームページ、トピック詳細 | 編集者が選んだ記事 |
| **最近更新** | ホームページ | 直近で更新された記事 |
| **人気記事** | ホームページ（将来） | アクセス数ベース（プライバシー考慮） |
| **関連記事** | 記事ページ | `concept-relations.yml` + トピック共通度 |
| **学習パス進捗** | 学習パス詳細 | ユーザーの進捗表示 |

### 4.4 検討事項

| 項目 | 質問 | 決定予定 |
| :--- | :--- | :--- |
| **人気記事** | アクセス数を収集するか？プライバシーとの兼ね合い | Phase 5以降 |
| **検索アナリティクス** | どこまで収集するか？ | 検索実装時 |
| **レコメンデーション** | アルゴリズムによる推薦は行わない（LEARNING_FOUNDATIONS準拠）→ 代わりに何を提供するか？ | Phase 5以降 |

> **参照**: 検索技術の詳細は [ARCHITECTURE_DECISION_RECORD.md](../../engine/technical/ARCHITECTURE_DECISION_RECORD.md) を参照。

| 項目 | 方針 |

| **学習パスナビ** | 左サイドバー | 記事上下部に簡易版 + ボトムシート展開 |
| **検索** | ヘッダー内 | アイコン → フルスクリーンモーダル |
| **フッター** | 複数列 | 単一列（アコーディオン検討） |

詳細は [ARTICLE_PAGE_DESIGN.md](./ARTICLE_PAGE_DESIGN.md) セクション10を参照。

| 要素 | 対応 |
| :--- | :--- |
| **レイアウト** | 3カラム → 1カラム |
| **目次** | スティッキー目次バー（~40px）+ アコーディオン |
| **学習パスナビ** | 簡易版（パス名 + 全体感 + 前後リンク）+ ボトムシート展開 |
| **本文** | 全幅表示、適切な行長（読みやすさ） |
| **コードブロック** | 水平スクロール |
| **テーブル** | 水平スクロール or カード形式 |
| **画像** | レスポンシブ画像（srcset） |

## 6. アクセシビリティ考慮

> **ステータス**: 🔄 ドラフト
> **準拠目標**: WCAG 2.2 AA

### 6.1 ナビゲーションのアクセシビリティ

| 要素 | 要件 |
| :--- | :--- |
| **キーボード操作** | すべてのナビゲーションがキーボードで操作可能 |
| **フォーカス表示** | フォーカスリングを常に可視（`outline: none` 禁止） |
| **スキップリンク** | ヘッダー直後に「本文へスキップ」リンク |
| **ARIA ランドマーク** | `<nav>`、`<main>`、`<footer>` 等の適切な使用 |

### 6.2 検索のアクセシビリティ

| 要素 | 要件 |
| :--- | :--- |
| **検索モーダル** | Escで閉じる、フォーカストラップ |
| **検索結果** | ライブリージョン（`aria-live`）で結果件数を通知 |
| **フィルター** | チェックボックス/ラジオボタンは適切なラベル付け |

### 6.3 記事コンテンツのアクセシビリティ

| 要素 | 要件 |
| :--- | :--- |
| **見出し構造** | `<h1>` から順に適切な階層 |
| **画像** | 意味のある `alt` 属性（装飾は空文字） |
| **コードブロック** | スクリーンリーダーで読み上げ可能 |
| **用語定義（Toggletip）** | `button` + `aria-expanded` |

> **参照**: アクセシビリティの詳細要件は `ACCESSIBILITY_GUIDELINES.md`（予定）を参照。
