# コンテンツガバナンス (Content Governance)

本ドキュメントでは、Isidoricaにおけるコンテンツの執筆、レビュー、出典管理に関するガバナンスルールを定義する。

> **翻訳関連**: 多言語対応・翻訳ワークフローについては [TRANSLATION_DESIGN.md](./TRANSLATION_DESIGN.md) を参照。理論的基盤は [TRANSLATION_FOUNDATIONS.md](../shared/TRANSLATION_FOUNDATIONS.md) を参照。

---

## 1. 出典管理と信頼性 (Source Management & Reliability)

### 1.1 基本原則

コンテンツの信頼性は、執筆者の権威ではなく、提示された出典の検証可能性によって担保される（**Source Supremacy**）。すべての知識主張および使用されるデータセットは、Isidoricaが定める信頼性基準を満たす出典に基づかなければならない。

### 1.2 信頼性階層 (Reliability Tiers)

出典およびデータセットは、その信頼性に応じて階層化（Tier）される。執筆者は常に最上位のTierに属する情報を優先的に使用する義務を負う。各Tierの詳細定義および使用可能な範囲は [CONTENT_POLICY.md](../shared/CONTENT_POLICY.md) にて定める。

### 1.3 合成データの取り扱い

教育的説明およびシミュレーションを目的とした「合成データ（Synthetic Data）」の使用は許容される。ただし、**架空のデータであることを意図的に隠蔽し、事実として誤認させる行為は、重大な認識的背信行為（Epistemic Betrayal）として禁止する**。すべての合成データには、その性質を明示する注記が必須である。

### 1.4 違反への措置

出典の偽装、不適切なTierの意図的な使用、合成データの隠蔽など、情報の信頼性を著しく損なう行為が確認された場合、当該コンテンツの即時削除に加え、関与した執筆者およびレビュワーの権限剥奪を含む厳格な措置を講じる。

### 1.5 編集の自己反省原則（Critical Self-Awareness）

Isidoricaは**批判的メディアリテラシー（Critical Media Literacy）**の対象であることを認識する。

| 原則 | 内容 |
| :--- | :--- |
| **非中立性の自覚** | すべてのコンテンツ選択・構成・表現には価値判断が含まれる。「中立的な情報提供」は存在しない |
| **権力構造の認識** | 何を含め、何を除外するかは権力の行使である。この構造を自覚し、可能な限り透明化せよ |
| **代替視点の開示** | 論争的トピックにおいては、採用しなかった視点・解釈も可能な限り言及せよ |
| **自己批判の継続** | Isidorica自身がCMLの批判対象であることを常に認識し、外部からの批判を受け入れる姿勢を維持せよ |

> **参照**: Kellner, D. & Share, J. (2007). *Critical Media Literacy, Democracy, and the Reconstruction of Education.*

## 2. 言語政策と公平性 (Language Policy & Equity)

Isidoricaは単一言語中心主義を排し、言語間の認識的公平性（Epistemic Equity）を追求する。

### 2.1 コンテンツの等価性

すべての言語版は、情報的価値において等価でなければならない。翻訳は単なる言語の変換ではなく、知識の再構築プロセスとして扱われる。

### 2.2 原典への敬意

出典参照においては、常に原典（Original Source）へのアクセス可能性を確保しつつ、読者の言語によるアクセス権も保障する二重構造（Dual Structure）を維持する。

> 詳細な運用設計については [TRANSLATION_DESIGN.md](./TRANSLATION_DESIGN.md) を参照。

## 3. 変更管理と承認権限 (Change Management & Authority)

### 3.1 変更提案権 (Right to Propose)

いかなる者も、所定の手続きを通じてコンテンツの追加・修正・削除を提案する権利を有する。

### 3.2 承認と統合 (Approval & Integration)

すべての変更提案は、独立した第三者（レビュワー）による検証と承認を経なければ、公式コンテンツとして統合（マージ）されない。自己承認による変更は禁止される。

### 3.3 緊急措置権 (Emergency Authority)

法的リスク、セキュリティ脅威、または著しい倫理違反が確認された場合、Core Maintainerは通常の承認プロセスを省略し、当該コンテンツを即時停止（削除または非公開化）する権限を行使できる。

## 4. 査読と品質保証 (Peer Review & Quality Assurance)

### 4.1 査読の義務 (Mandatory Review)

すべてのコンテンツは、公開前に最低1名の有資格レビュワーによる査読を通過しなければならない。査読を経ない公開（Unreviewed Publication）は、緊急措置を除き一切認められない。

### 4.2 拒否権 (Veto Power)

レビュワーは、コンテンツがIsidoricaの定める品質基準（信頼性、中立性、教育的価値）を満たさないと判断した場合、その公開を拒否する権限を持つ。この判断に対し、執筆者は対等な立場で異議を申し立てることができる。

### 4.3 利益相反の開示 (Conflict of Interest)

執筆者およびレビュワーは、特定のトピックに関して個人的・経済的・政治的な利益相反がある場合、それを自主的に開示し、必要に応じて当該プロセスから辞退する義務を負う。

---

## 5. 訂正と説明責任 (Correction & Accountability)

### 5.1 訂正の義務 (Duty to Correct)

Isidoricaは「無謬性」を主張しない。コンテンツに誤りが発見された場合、開発チームはそれを迅速かつ透明性を持って訂正する義務を負う。既知の誤りを放置することは、その誤りの発信者と同罪とみなされる。

### 5.2 修正履歴の透明性 (Transparency of Revision)

誤りの訂正は、原則として削除ではなく修正（Revision）として行われ、修正履歴が誰でも検証可能な状態で保存されなければならない。過去の誤りを隠蔽する行為（ステルス修正）は、信頼性を損なう行為として禁止される。

### 5.3 陳腐化への対応 (Obsolescence Management)

知識は時間とともに陳腐化する。長期間更新されていないコンテンツに対し、コミュニティは「陳腐化警告（Obsolescence Warning）」を付与し、読者に注意を促す権利を有する。

## 10. 品質基準 (Quality Standards)

### 10.1 必須要件

| 要件 | 内容 |
| :--- | :--- |
| **レビュー必須** | PRマージ前にレビュワーが確認（必須） |
| **出典必須** | 検証可能な出典がなければ公開しない |
| **荒らし対策** | IPレート制限、クライアント検証、レビュワーによるフィルタリング |
| **正確性** | 事実に基づく記述、誤解を招く表現の排除 |
| **中立性** | 特定の思想・立場に偏らない |
| **アクセシビリティ** | WCAG 2.2 AA準拠、スクリーンリーダー対応 |

### 10.2 推奨事項

| 推奨 | 内容 |
| :--- | :--- |
| **平易な表現** | 専門用語には説明を付与 |
| **構造化** | 見出し、リスト、表を活用した読みやすい構造 |
| **公開情報** | 著者名なし、最終更新日のみ表示 |
| **内部管理** | GitHubアカウントまたはAgoraセッションIDで追跡 |
| **国際的な出典** | 可能な限り複数言語で入手可能な出典を優先 |

### 10.3 画像・図表の権利処理

画像・図表を含むすべての非テキストメディアについて、執筆者は以下の責任を負う。

| 役割 | 責任 |
| :--- | :--- |
| **執筆者** | 適切な許諾の取得、およびライセンス条件（CC BY等）の遵守 |
| **レビュワー** | 許諾およびライセンス適合性の確認 |

※ いかなる場合も、著作権を侵害するメディアの掲載は認められない。

## 11. 紛争解決 (Dispute Resolution)

### 11.1 根本原則

紛争解決は**「誰が正しいか」ではなく「何が根拠か」**で判断する。すべての議論は公開された検証可能な出典に基づいて行われなければならない。執筆者自身への批判（人格攻撃）は厳に禁止される。

### 11.2 解決プロセス

紛争は以下の段階を経て解決される。

1.  **当事者間協議**: 出典に基づく相互検証。
2.  **コミュニティ熟議 (Agora)**: 第三者を交えた公開討論。
3.  **最終裁定**: Core Maintainerによる、証拠の優位性に基づく最終判断。

### 11.3 再審請求権 (Right to Appeal)

裁定は原則として最終決定であるが、**新しい有効な出典**が提示された場合に限り、いつでも再審を請求する権利が保障される。これはIsidoricaが権威ではなく証拠に従うことを意味する。

### 11.4 複数解釈の尊重

学術的に複数の有力な説が存在する場合、いずれか一つを「正解」として定めるのではなく、両論を公平に併記し、読者に判断を委ねることを原則とする。

## 12. ガバナンス維持 (Governance Maintenance)

### 12.1 定期レビュー

本ガバナンス文書および関連するポリシー文書は、プロジェクトの成長と環境変化に適応するため、Core Maintainerによって定期的に（原則として年1回以上）見直されなければならない。

### 12.2 エンドオブライフ

技術選定や運用ルールは変化するが、Isidoricaの「知識のアクセシビリティ」と「認識的公平性」に対するコミットメントは恒久的なものである。
