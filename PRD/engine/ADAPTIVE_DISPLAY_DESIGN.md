## 1. 概要

### 1.1 本文書の目的

本文書は、PRODUCT_DESIGN §2で定義された4次元読者モデルに基づき、**P（アクセス目的）**と**T（時間制約）**次元への具体的な表示適応を定義する。

> **注記**: K（知識状態）次元とM（メタ学習スキル）次元の適応は [LEARNER_STATE_MODEL.md](../shared/LEARNER_STATE_MODEL.md)で定義。

### 1.2 設計思想の転換

| 旧モデル（廃止） | 新モデル（採用） |
|:---|:---|
| Explorer/Learner/Scholar という**固定的な読者タイプ** | K, M, P, T という**動的な4次元**で読者状態を捉える |
| 読者をタイプに分類しトグルで切り替え | 次元の組み合わせに応じてシステムが動的に適応 |

### 1.3 旧モデルとの対応関係

| 旧モデル | 4次元モデルでの表現 |
|:---|:---|
| **Explorer** | P = P_reception, T = T_quick |
| **Learner** | P = P_cognitive, T = T_extended |
| **Scholar** | K = K₂, P = P_cognitive または P_reception |

## 2. P次元（アクセス目的）への適応

### 2.1 P次元の値と表示モード

| P値 | 典型的な問い | 表示モード | 優先表示要素 |
|:---|:---|:---|:---|
| **P_reception** | 「これは何？」 | **参照モード** | TL;DR、定義、即時回答 |
| **P_production** | 「これはどう使う？」 | **産出支援モード** | 用例、コロケーション、文法 |
| **P_cognitive** | 「なぜこうなる？」 | **学習モード** | Phase 1-6構造、詳細説明 |
| **P_solving** | 「どうすればできる？」 | **問題解決モード** | 手順、適用例、FAQ |

### 2.2 各モードの詳細設計

#### P_reception（参照モード）

| 項目 | 設計 |
|:---|:---|
| **目的** | 即座に意味を知りたい |
| **トリガー** | 検索からの流入、`[[Term]]` クリック、T_quick との組み合わせ |
| **表示構成** | TL;DR（1-2文）→ 定義 → 「詳しく学ぶ」リンク |
| **Phase構造** | 非表示（必要なら展開可能） |
| **Laboratory** | 非表示（または縮小表示） |

#### P_cognitive（学習モード）

| 項目 | 設計 |
|:---|:---|
| **目的** | 体系的に概念を理解したい |
| **トリガー** | 学習パスからの遷移、「詳しく学ぶ」クリック、T_extended との組み合わせ |
| **表示構成** | Phase 1-6 の完全構造 |
| **Phase構造** | 完全表示、ナビゲーション付き |
| **Laboratory** | 完全にインタラクティブ |

#### P_production（産出支援モード）

| 項目 | 設計 |
|:---|:---|
| **目的** | 文章作成中に正しい使い方を知りたい |
| **トリガー** | エディタからの参照、明示的選択 |
| **表示構成** | 用例セクション優先、コロケーション、文法的注意点 |
| **Phase構造** | Phase 6（応用・転移）を優先表示 |
| **Laboratory** | 必要に応じて表示 |

#### P_solving（問題解決モード）

| 項目 | 設計 |
|:---|:---|
| **目的** | 具体的な問題を解決したい |
| **トリガー** | 問題文からの参照、FAQ経由 |
| **表示構成** | 手順、適用例、よくある間違い、トラブルシューティング |
| **Phase構造** | Phase 3-4（機序・確認問題）を優先表示 |
| **Laboratory** | 操作可能な状態で表示 |

## 3. T次元（時間制約）への適応

### 3.1 T次元の値と詳細度

| T値 | 典型的状況 | 詳細度 | 要約度 |
|:---|:---|:---|:---|
| **T_quick** | 会議中、文章読解中 | 低（最小限） | 高（TL;DR中心） |
| **T_extended** | 自習時間、研究中 | 高（完全版） | 低（詳細表示） |

### 3.2 T_quick への適応

| 要素 | T_quick での扱い |
|:---|:---|
| **TL;DR** | **必須表示**、最優先 |
| **本文** | デフォルト折りたたみ、展開可能 |
| **Phase構造** | ナビゲーションのみ表示、内容は折りたたみ |
| **Laboratory** | サムネイルのみ、クリックで展開 |
| **出典** | インライン最小化、参考文献リストは折りたたみ |

### 3.3 T_extended への適応

| 要素 | T_extended での扱い |
|:---|:---|
| **TL;DR** | 表示（スキップ可能） |
| **本文** | 完全展開 |
| **Phase構造** | 完全展開、ナビゲーション付き |
| **Laboratory** | 完全インタラクティブ |
| **出典** | 詳細表示 |

## 4. Phase構造（維持）

### 4.1 6 Phase モデル

Phase構造は全モードで**構造として維持**するが、表示の詳細度をP/Tに応じて調整する。

| Phase | 内容 | P_cognitive（全表示） | P_reception（圧縮） |
|:---|:---|:---|:---|
| **Phase 1** | 予測誤差の喚起 | 完全表示 | 圧縮版 or 省略 |
| **Phase 2** | 定義 | 完全表示 | **優先表示** |
| **Phase 3** | 機序 | 完全表示 | 折りたたみ |
| **Phase 4** | 確認問題 | 完全表示 | 省略 |
| **Phase 5** | 文脈・歴史 | 完全表示 | 折りたたみ |
| **Phase 6** | 応用・転移 | 完全表示 | 折りたたみ |

### 4.2 離脱防止策（旧設計から継承）

各Phaseでの離脱リスクと対策（P_cognitive モード向け）：

| Phase | リスク | 原因 | 対策 |
|:---|:---|:---|:---|
| **Phase 2** | 高 | 抽象的で退屈 | Phase 1の「驚き」を再利用した具体例 |
| **Phase 3** | 高 | 複雑で混乱 | 段階的な足場かけ、コンテキスト依存ヒント |
| **Phase 4** | 中 | 失敗恐怖 | 失敗を「発見」として再フレーミング |
| **Phase 5** | 中 | 関連性不明 | 「なぜ重要か」を冒頭で明示 |
| **Phase 6** | 低 | 達成感で完結 | 次の学習パス、Agoraへの導線 |

## 5. 状態判定と切り替え

### 5.1 P次元の判定

| 判定方法 | 条件 | 推定されるP |
|:---|:---|:---|
| **流入元** | 検索エンジンから | P_reception |
| **流入元** | 学習パスから | P_cognitive |
| **流入元** | `[[Term]]` クリック | P_reception |
| **明示的選択** | タブ/ドロップダウン | ユーザー指定 |
| **過去の行動** | 同セッションで学習パス利用 | P_cognitive を維持 |

### 5.2 T次元の判定

| 判定方法 | 条件 | 推定されるT |
|:---|:---|:---|
| **デフォルト** | 検索からの初回訪問 | T_quick |
| **デフォルト** | 学習パスからの遷移 | T_extended |
| **明示的選択** | 「時間がない」チェック | T_quick |
| **明示的選択** | 「詳しく学ぶ」クリック | T_extended |
| **過去の行動** | 同セッションで長時間滞在 | T_extended |

### 5.3 UI切り替えの原則

| 原則 | 説明 |
|:---|:---|
| **強制しない** | モード切り替えを強制せず、ナッジで誘導 |
| **可逆的** | いつでも切り替え可能 |
| **記憶** | localStorageでユーザー設定を保持 |
| **漸進的開示** | 圧縮表示→必要に応じて展開 |

## 6. 技術設計

### 6.1 データ属性

```html
<article 
  data-p-mode="cognitive" 
  data-t-mode="extended"
  data-k-state="K1"
  data-m-level="high">
  ...
</article>
```

### 6.2 CSS による表示切り替え

```css
/* P_reception モード */
[data-p-mode="reception"] .phase-content { display: none; }
[data-p-mode="reception"] .tldr { display: block; }

/* T_quick モード */
[data-t-mode="quick"] .laboratory { display: none; }
[data-t-mode="quick"] .detailed-content { display: none; }
```

### 6.3 localStorage

| キー | 値 | デフォルト |
|:---|:---|:---|
| `isidorica_p_default` | `reception` \| `cognitive` \| `production` \| `solving` | 流入元から推定 |
| `isidorica_t_default` | `quick` \| `extended` | 流入元から推定 |
