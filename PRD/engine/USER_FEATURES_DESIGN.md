# ユーザー機能設計 (User Features Design)

本ドキュメントでは、Isidoricaにおけるユーザー向け機能の設計方針を定める。全ての機能は [LEARNING_FOUNDATIONS.md](../shared/LEARNING_FOUNDATIONS.md) に基づく教育理論と整合性を持つよう設計される。

**ステータス**: ドラフト

---

## 1. 基本方針

### 1.1 アカウント不要の原則

| 原則 | 説明 |
| :--- | :--- |
| **全機能アカウント不要** | 記事閲覧、Laboratory、学習進捗保存など、全ての学習機能はアカウントなしで利用可能 |
| **LocalStorage活用** | 学習データはブラウザのLocalStorageに保存。個人情報を収集しない |
| **アカウントはオプション** | 複数デバイス同期が必要な場合のみアカウント作成（13歳以上） |

### 1.2 理論的基盤

| 理論 | 適用 |
| :--- | :--- |
| **自己決定理論（SDT）** | 自律性（Autonomy）、有能感（Competence）、関係性（Relatedness）を満たす設計 |
| **予測処理理論** | 予測誤差（驚き）を学習の駆動力として活用 |
| **メタ認知** | 学習者が自己の理解度をモニタリングできる機能 |
| **認知負荷理論** | 外在的負荷を最小化し、学習関連負荷にリソースを集中 |

## 2. 機能カタログ

### 2.1 進捗・学習管理

#### 2.1.1 学習進捗の同期

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 複数デバイス間での学習進捗の同期 |
| **アカウント** | ✅ 必要（13歳以上） |
| **優先度** | 高（アカウント機能のコア） |
| **理論的根拠** | - |
| **実装方針** | クラウドストレージ（Cloudflare KV等）で進捗データを保存 |

#### 2.1.2 学習履歴

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 過去に読んだ記事の一覧表示 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 高 |
| **理論的根拠** | **メタ認知（3.7）**: 自己の学習をモニタリングする「鏡」 |
| **実装方針** | 閲覧日時、完了状態、滞在時間を記録 |

#### 2.1.3 学習キュー（旧: ブックマーク）

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 学習予定の記事を管理 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 中 |
| **理論的根拠** | **認知負荷（3.5）**: 学習計画を外在化し、ワーキングメモリを解放 |
| **設計上の注意** | 「後で読む」ではなく**「学習キュー」**として設計。未読を可視化しプレッシャーを与えない |
| **実装方針** | キュー形式（FIFO）で表示。古いものを優先表示 |

#### 2.1.4 学習ノート

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 記事ごとにメモを追加できる機能 |
| **アカウント** | ❌ 不要（LocalStorage）/ 同期はアカウント要 |
| **優先度** | 高 |
| **理論的根拠** | **自己説明（Phase 4）**: 自分の言葉でまとめることがスキーマ構築に有効 |
| **設計上の注意** | 空白のメモ欄ではなく、**「この記事で最も重要なことは？」**などのプロンプトを提示 |
| **実装方針** | Markdown対応、記事IDに紐付けて保存 |

#### 2.1.5 Learning Path進捗

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 学習パスの完了状況を可視化 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 高 |
| **理論的根拠** | **有能感（3.3 SDT）**: 進捗の可視化がCompetence欲求を満たす |
| **設計上の注意** | 小さな達成を祝うマイクロフィードバック（例: 進捗バーのアニメーション） |
| **実装方針** | 各記事の完了状態、全体の進捗率を表示 |

#### 2.1.6 知識状態トラッキング（将来実装）

| 項目 | 内容 |
| :--- | :--- |
| **概要** | ユーザーが学習済みの概念を追跡し、前提知識の充足をチェック |
| **アカウント** | ✅ 必要（学習状態の永続化） |
| **優先度** | 中（Phase 3以降） |
| **理論的根拠** | **動的意味論（9.1）**: ユーザーの知識状態 $K$ をモデル化し、前提（Presupposition）の充足を検証 |
| **設計上の注意** | **強制ではなく提案**。前提未充足時は「この記事を先に読むことをお勧めします」と提示。ブロックはしない |
| **実装方針** | 学習済み記事IDの集合 $K$ をアカウントに保存。記事の `requires` フィールドと照合 |

> **参照**: 理論的背景は [LEARNING_FOUNDATIONS.md 9.1節](../shared/LEARNING_FOUNDATIONS.md) を参照。

### 2.2 パーソナライゼーション

#### 2.2.1 関連記事の探索（旧: おすすめ記事）

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 学習履歴に基づく関連記事の提示 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 中 |
| **理論的根拠** | **自律性（3.3 SDT）**: アルゴリズムによる「誘導」ではなく「探索の支援」 |
| **設計上の注意** | 「あなたへのおすすめ」ではなく**「関連トピック」**として提示。ユーザーが能動的に選択 |
| **実装方針** | 記事のトピック、閲覧履歴に基づくシンプルなルールベース |

#### 2.2.2 難易度設定

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 説明レベルの調整（入門/中級/上級） |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 中 |
| **理論的根拠** | **ZPD（3.6 Vygotsky）**: 学習者に合った難易度が最適学習を促進 |
| **設計上の注意** | **ユーザーが手動で選択**。自動調整しない（自律性の尊重） |
| **実装方針** | グローバル設定 + 記事ごとのオーバーライド可能 |

#### 2.2.3 表示設定

| 項目 | 内容 |
| :--- | :--- |
| **概要** | フォントサイズ、テーマ、行間等の調整 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 低 |
| **理論的根拠** | **認知負荷（3.5）**: 個人差への対応 |
| **実装方針** | システム設定を尊重しつつ、ユーザーによるオーバーライドを許可 |

### 2.3 インタラクション

#### 2.3.1 Laboratory成果保存

| 項目 | 内容 |
| :--- | :--- |
| **概要** | Laboratoryで書いたコードの保存 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 高 |
| **理論的根拠** | **エナクティビズム（3.2）**: 行為の結果を保存することで振り返りが可能に |
| **実装方針** | 記事IDとLabID単位で保存。履歴機能（複数バージョン） |

#### 2.3.2 クイズ・理解度チェック

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 記事末の確認問題 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 高 |
| **理論的根拠** | **Phase 4 Metacognition**: JOL（学習判断）較正の核心機能 |
| **設計上の注意** | **「正解/不正解」だけでなく「なぜ間違えたか」「正しい考え方」のフィードバック**を提供 |
| **実装方針** | 選択式・記述式。スコアと回答履歴を保存 |

#### 2.3.3 間違いノート

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 間違えた問題の記録と分析 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 高 |
| **理論的根拠** | **予測誤差（3.1）**: 間違い＝予測誤差。これを学習の起点として活用 |
| **設計上の注意** | 単なる問題リストではなく、**「なぜ間違えたか」の自己分析を促すUI** |
| **実装方針** | 間違えた問題、選択した回答、正解、自己分析メモを記録 |

## 3. コンセプトモデルに基づく新規機能提案

### 3.1 予測誤差の活用

#### 3.1.1 「驚き」ログ

| 項目 | 内容 |
| :--- | :--- |
| **概要** | Laboratoryで「予想と違った！」瞬間を記録 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 中 |
| **理論的根拠** | **予測誤差（3.1）**: 驚きの瞬間を可視化し、学習の起点として活用 |
| **実装方針** | Laboratory内に「驚いた！」ボタン。押した瞬間の状態をスナップショット保存 |

### 3.2 メタ認知支援

#### 3.2.1 JOL（学習判断）履歴

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 「分かった」と思った後、Quizで正解したか追跡 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 中 |
| **理論的根拠** | **メタ認知（3.7）**: 過信（Overconfidence）の傾向を可視化 |
| **実装方針** | 記事完了時にJOL評価（0-100%）を記録。Quiz結果と比較して較正精度を表示 |

#### 3.2.2 「説明してみる」機能

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 記事を読んだ後、自分の言葉で説明を書く欄 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 高 |
| **理論的根拠** | **自己説明（Phase 4）**: 説明することで理解の穴が見える |
| **設計上の注意** | **評価・添削機能は将来実装**。まずは書く行為自体を促す |
| **実装方針** | 記事末に「この概念を友人に説明するなら？」欄を配置 |

### 3.3 動機づけ支援

#### 3.3.1 「なぜ学ぶか」リマインダー

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 記事冒頭で「これを学ぶと何ができるか」を明示 |
| **アカウント** | - |
| **優先度** | 高（記事設計に組み込み） |
| **理論的根拠** | **価値（3.4 Pekrun）**: 学習の価値を認識することで享楽（Enjoyment）を維持 |
| **実装方針** | 記事Frontmatterに`learning_outcomes`フィールドを追加 |

#### 3.3.2 「間違いからの学び」ダッシュボード

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 間違いのパターン分析、弱点の可視化 |
| **アカウント** | ❌ 不要（LocalStorage） |
| **優先度** | 中 |
| **理論的根拠** | **予測誤差（3.1）**: 間違いを恐れるのではなく、学習の糧として捉える |
| **実装方針** | トピック別の正答率、間違いやすい概念の可視化 |

## 4. 採用を慎重に検討する機能

以下の機能は、コンセプトモデルとの整合性に懸念があり、採用する場合は慎重な設計が必要。

### 4.1 バッジ/実績

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 学習達成のゲーミフィケーション |
| **懸念** | **外発的動機づけ（3.3 SDT）**: バッジ獲得が目的化すると、内発的動機が阻害される |
| **採用基準** | 採用する場合は以下の条件を満たすこと |
| **条件1** | 「意外な達成」に限定（予測可能なバッジは外発的動機を強化） |
| **条件2** | 主要UIに表示しない（プロフィールページにのみ） |
| **条件3** | 他者との比較要素を排除 |

### 4.2 公開プロフィール

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 学習実績の公開 |
| **懸念** | **社会的比較**: 他者との比較は不安を誘発するリスク |
| **採用基準** | 採用する場合は以下の条件を満たすこと |
| **条件1** | 完全にオプトイン（デフォルトは非公開） |
| **条件2** | ランキング・順位表示なし |
| **条件3** | 学習量ではなく学習トピックの可視化に限定 |

## 6. 技術的考慮事項

### 6.1 LocalStorage

| 項目 | 内容 |
| :--- | :--- |
| **容量制限** | 約5MB（ブラウザによる） |
| **対策** | 古いデータの自動アーカイブ、IndexedDBへの移行検討 |
| **データ構造** | 記事ID、タイムスタンプ、状態を効率的に保存 |

### 6.2 データ同期（アカウント機能）

| 項目 | 内容 |
| :--- | :--- |
| **バックエンド** | Cloudflare Workers + KV / D1 |
| **同期戦略** | オフラインファースト、差分同期 |
| **コンフリクト解決** | Last Write Wins（将来的にはマージ） |

### 6.3 プライバシー

| 項目 | 内容 |
| :--- | :--- |
| **LocalStorage** | 個人情報を収集しない |
| **アカウント** | 最小限のデータのみ（メールアドレス、進捗）、GDPR/CCPA準拠 |
